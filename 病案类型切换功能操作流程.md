## 📋 完整操作流程

### 场景1: 新建病案（选择不同类型）

#### 步骤1: 进入病案列表
```
登录系统 → 住院管理 → 住院患者 → 选择患者 → 点击"病案列表"
```

#### 步骤2: 点击新增病案下拉菜单
```
点击"新增病案"按钮右侧的下拉箭头
```

#### 步骤3: 选择病案类型

**选项A: 入院记录**
```
下拉菜单 → 选择"入院记录"
↓
打开 AdmissionRecordForm.vue 表单
显示字段：主诉、现病史、既往史、生命体征、体格检查等
```

**选项B: 首次病程记录**
```
下拉菜单 → 选择"首次病程记录"
↓
打开 ProgressRecordForm.vue 表单
显示字段：病例特点、初步诊断、诊断依据、鉴别诊断等
```

**选项C: 出院记录**
```
下拉菜单 → 选择"出院记录"
↓
打开 DischargeRecordForm.vue 表单
显示字段：入院情况、诊疗经过、出院诊断、出院医嘱等
```

**选项D: 手术记录**
```
下拉菜单 → 选择"手术记录"
↓
打开 OperationRecordForm.vue 表单
显示字段：手术名称、手术者、麻醉方式、手术经过等
```

#### 步骤4: 填写表单
- 患者信息自动填充（不可修改）
- 病案号自动等于住院号
- 填写该类型特有的字段
- ICD-10编码可使用"检查编码"功能

#### 步骤5: 保存
```
点击"确定" → 提示"创建成功" → 列表刷新
```

#### 验证结果
- ✅ 列表中显示新病案
- ✅ "病案类型"列显示对应标签（带颜色）
  - 入院记录: 🔵 蓝色
  - 病程记录: 🟢 绿色
  - 出院记录: 🟠 橙色

## 🔍 核心实现逻辑

### 1. 新建病案时的类型选择

**代码位置**: `handleCreateByType` 函数（第352-377行）

```typescript
const handleCreateByType = (type: string) => {
  if (!patientId.value) {
    ElMessage.warning('请先选择患者')
    return
  }
  
  currentRecord.value = null
  
  switch (type) {
    case 'admission':
      showAdmissionForm.value = true  // 打开入院记录表单
      break
    case 'progress':
      showProgressForm.value = true   // 打开病程记录表单
      break
    case 'discharge':
      showDischargeForm.value = true  // 打开出院记录表单
      break
    case 'operation':
      showOperationForm.value = true  // 打开手术记录表单
      break
    default:
      dialogVisible.value = true      // 默认使用通用表单
  }
}
```

### 2. 编辑病案时的自动识别

**代码位置**: `handleEdit` 函数（第394-415行）**← 刚刚修复！**

```typescript
const handleEdit = (row: any) => {
  currentRecord.value = row
  
  // 根据病案类型打开对应的表单
  switch (row.recordType) {
    case 'admission':
      showAdmissionForm.value = true  // 打开入院记录表单
      break
    case 'progress':
      showProgressForm.value = true   // 打开病程记录表单
      break
    case 'discharge':
      showDischargeForm.value = true  // 打开出院记录表单
      break
    case 'operation':
      showOperationForm.value = true  // 打开手术记录表单
      break
    default:
      // 未知类型或其他类型，使用通用表单
      dialogVisible.value = true
  }
}
```

## 🎯 常见问题

### Q1: 为什么编辑时表单没有切换？
**A**: 之前的 `handleEdit` 函数没有根据 `recordType` 判断，已修复！现在会自动识别并打开对应表单。

### Q2: 如何知道当前病案是什么类型？
**A**: 查看列表中的"病案类型"列，有彩色标签显示。

### Q3: 如果我选择了错误的类型怎么办？
**A**: 新建时可以重新选择；编辑时类型由数据库决定，无法更改（这是正确的行为）。

### Q4: 通用表单还会用到吗？
**A**: 会，用于：
- 未知类型的病案
- 向后兼容旧数据
- 特殊场景需求

### Q5: 数据是如何存储的？
**A**: 所有数据存储在 `content` JSON 字段中，不同类型有不同的JSON结构：
- 入院记录: `{ "admissionRecord": {...} }`
- 病程记录: `{ "progressRecord": {...} }`
- 出院记录: `{ "dischargeRecord": {...} }`
- 手术记录: `{ "operationRecord": {...} }`
