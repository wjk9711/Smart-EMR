const qualityControlDiagnosis = (index: number) => {
  const diagnosis = dischargeDiagnoses.value[index]
  const result = qcDiagnosis(diagnosis.name, diagnosis.code)
  
  if (result.isValid) {
    ElMessage.success(result.message)
  } else {
    ElMessage.warning({ message: result.message, duration: 5000 })
    
    if (result.suggestedCode) {
      ElMessageBox.confirm(...).then(() => {
        diagnosis.code = result.suggestedCode!
        ElMessage.success('已自动填充建议编码')
      })
    }
  }
}

## 📊 功能对比

### 与教师审核质控的区别

| 对比项 | 病案首页质控（新功能） | 教师审核质控（原有） |
|-------|---------------------|-------------------|
| **使用场景** | 医生填写病案时 | 教师/质控员审核时 |
| **目的** | 快速检查编码正确性 | 全面审核病案质量 |
| **检查内容** | 仅检查疾病/手术编码 | 完整性、规范性、逻辑性等 |
| **复杂度** | 简单快捷，一键操作 | 复杂详细，需填写意见 |
| **反馈方式** | 直接给出建议编码 | 需要详细审核说明 |
| **自动化程度** | 高（自动建议+一键应用） | 低（人工判断） |
| **响应时间** | < 1秒 | 可能需要几分钟 |
| **适用人群** | 所有医生 | 教师/质控员 |

## ⚠️ 注意事项

### 1. 编码数据库限制
- 当前仅包含约60条示例编码
- 实际使用需要导入完整的ICD-10和ICD-9-CM-3数据库
- 建议从国家卫健委获取标准编码库

### 2. 网络依赖
- 当前为纯前端实现，无需后端支持
- 未来如需动态更新编码库，需要后端API支持

### 3. 浏览器兼容性
- 支持现代浏览器（Chrome, Firefox, Edge, Safari）
- 不支持IE11及以下版本

### 4. 性能考虑
- 单次质控响应时间 < 1秒
- 编码库在前端内存中，查询速度快
- 大量数据时建议优化为后端查询

## 🎉 总结

### 核心价值
1. **提高效率**：医生填写病案时快速验证编码，减少返工
2. **保证质量**：确保疾病和手术编码的准确性
3. **降低门槛**：智能建议编码，降低编码知识要求
