# 病案预览内容不显示问题修复报告

## 📋 问题描述

用户反馈：**在病案首页预览以及预览全部病案中，内容均不显示**

---

## 🔍 问题分析

### 根本原因

**PreviewContent 组件没有接收 data prop** ❌

在 `HomePagePreview.vue` 和 `AdmissionRecordPreview.vue` 中：

1. **定义了内部组件 PreviewContent**：
   ```typescript
   const PreviewContent = defineComponent({
     props: ['data'],  // ← 定义了 data prop
     setup(props) {
       return {
         data: props.data,  // ← 期望接收 data
         // ...
       }
     },
     template: `
       <div>
         <!-- 模板中使用 data -->
         <span>{{ data.name || '-' }}</span>
       </div>
     `
   })
   ```

2. **但在模板中没有传递 data**：
   ```vue
   <!-- 错误写法 ❌ -->
   <PreviewContent />
   ```

3. **结果**：
   - `props.data` 为 `undefined`
   - 模板中所有 `data.xxx` 都显示为空或默认值 `-`
   - 预览界面看起来是空白的

---

## ✅ 解决方案

### 修复 PreviewContent 调用

在模板中正确传递 `data` prop：

**修改前** ❌：
```vue
<PreviewContent />
```

**修改后** ✅：
```vue
<PreviewContent :data="data" />
```

## 🎯 技术原理

### Vue 组件 Props 传递

在 Vue 3 中，子组件需要通过 props 接收父组件传递的数据：

```typescript
// 子组件定义
const ChildComponent = defineComponent({
  props: ['data'],  // 声明接收的 props
  setup(props) {
    return {
      data: props.data  // 在 setup 中返回，供模板使用
    }
  },
  template: `
    <div>{{ data.name }}</div>  // 模板中使用
  `
})
```

```vue
<!-- 父组件调用 -->
<ChildComponent :data="parentData" />  <!-- 必须传递 prop -->
```

### 常见错误

**错误1**: 忘记传递 prop
```vue
<ChildComponent />  <!-- ❌ data 为 undefined -->
```

**错误2**: prop 名称不匹配
```vue
<ChildComponent :info="parentData" />  <!-- ❌ 子组件期望的是 data -->
```

**正确做法**:
```vue
<ChildComponent :data="parentData" />  <!-- ✅ 名称匹配 -->
```

## 💡 预防措施

### 1. TypeScript 类型检查

使用 TypeScript 接口定义 props：

```typescript
interface PreviewContentProps {
  data: Record<string, any>
}

const PreviewContent = defineComponent<PreviewContentProps>({
  props: {
    data: {
      type: Object,
      required: true  // ← 标记为必需
    }
  },
  // ...
})
```

### 2. 运行时验证

在 setup 中添加验证：

```typescript
setup(props) {
  if (!props.data) {
    console.warn('PreviewContent: data prop is required!')
  }
  
  return {
    data: props.data
  }
}
```

### 3. 单元测试

编写测试用例验证 props 传递：

```typescript
import { mount } from '@vue/test-utils'
import HomePagePreview from './HomePagePreview.vue'

test('PreviewContent receives data prop', () => {
  const wrapper = mount(HomePagePreview, {
    props: {
      visible: true,
      recordData: {
        content: JSON.stringify({
          homePage: { name: '张三' }
        })
      }
    }
  })
  
  expect(wrapper.text()).toContain('张三')
})
```

## ✅ 总结

### 问题根源

- ❌ `PreviewContent` 组件定义了 `props: ['data']`
- ❌ 但在模板调用时没有传递 `:data="data"`
- ❌ 导致 `props.data` 为 `undefined`
- ❌ 所有内容显示为空或默认值 `-`

### 解决方案

- ✅ 在两处调用 `<PreviewContent>` 的地方添加 `:data="data"`
- ✅ 确保数据正确从父组件传递到子组件
- ✅ 恢复预览界面的正常显示

### 影响范围

- ✅ 病案首页单个预览功能恢复正常
- ✅ 病案首页在"预览全部病案"中正常显示
- ✅ 入院记录单个预览功能恢复正常
- ✅ 入院记录在"预览全部病案"中正常显示

### 经验教训

1. **Vue 组件 props 必须显式传递**
   - 即使子组件定义了 prop，父组件也必须传递
   
2. **TypeScript 不能完全防止这类错误**
   - 需要结合运行时检查和测试
   
3. **调试技巧**
   - 在子组件中添加 `console.log(props.data)` 快速定位问题
   - 使用 Vue Devtools 检查组件的 props
