# 住院号重复问题修复报告

## 问题描述

创建新患者时报错：
```
Duplicate entry '2024051001' for key 'inpatient_patients.inpatient_no'
```

**错误原因**：前端提交的住院号`2024051001`已存在于数据库中，违反了唯一约束。

## 问题分析

### 数据库约束
```sql
-- inpatient_patients表的inpatient_no字段有UNIQUE约束
ALTER TABLE `inpatient_patients` ADD UNIQUE KEY `inpatient_no` (`inpatient_no`);
```

### 问题场景

1. **管理员创建了患者A**，住院号为`2024051001`
2. **其他用户尝试创建患者B**，也输入了住院号`2024051001`
3. **数据库拒绝插入**，因为违反唯一约束

### 根本原因

- 前端允许用户手动输入住院号
- 没有实时验证住院号的唯一性
- 后端直接尝试插入，导致冲突

## 解决方案

### 策略：智能住院号处理

在后端创建患者时，采用以下逻辑：

```typescript
// 1. 如果前端未提供住院号 → 自动生成
// 2. 如果前端提供了住院号但已存在 → 自动生成新的
// 3. 如果前端提供了住院号且不存在 → 使用提供的住院号
```

### 实现代码

```typescript
export const createInpatientPatient = async (req: AuthRequest, res: Response) => {
  try {
    const currentUser = req.user!
    
    // 排除 sourcePatientId 和 inpatientNo
    const { sourcePatientId, inpatientNo: requestedInpatientNo, ...restData } = req.body
    
    // 处理住院号：如果未提供或已存在，则自动生成
    let finalInpatientNo = requestedInpatientNo
    
    if (!finalInpatientNo) {
      // 未提供住院号，自动生成
      finalInpatientNo = await generateUniqueInpatientNo()
    } else {
      // 检查住院号是否已存在
      const existing = await InpatientPatient.findOne({
        where: { inpatientNo: finalInpatientNo },
      })
      
      if (existing) {
        // 住院号已存在，自动生成新的
        console.log(`⚠️  住院号 ${finalInpatientNo} 已存在，自动生成新的住院号`)
        finalInpatientNo = await generateUniqueInpatientNo()
      }
    }
    
    const patientData = {
      ...restData,
      inpatientNo: finalInpatientNo,
      doctorId: currentUser.id,
      sourcePatientId: null,
    }
    
    const patient = await InpatientPatient.create(patientData)
    res.json({
      code: 200,
      data: patient,
      message: '创建成功',
    })
  } catch (err: any) {
    console.error('Create inpatient patient error:', err)
    return error(res, 500, err.message)
  }
}
```

### 关键特性

#### 1. 自动检测冲突
```typescript
const existing = await InpatientPatient.findOne({
  where: { inpatientNo: finalInpatientNo },
})

if (existing) {
  // 发现冲突，自动生成新的

}
```

#### 2. 友好提示
```typescript

```

#### 3. 保证唯一性
使用`generateUniqueInpatientNo()`函数确保生成的住院号全局唯一：
```typescript
async function generateUniqueInpatientNo(): Promise<string> {
  let inpatientNo = ''
  let exists = true
  
  while (exists) {
    const date = new Date()
    const dateStr = date.getFullYear().toString() + 
                   (date.getMonth() + 1).toString().padStart(2, '0') + 
                   date.getDate().toString().padStart(2, '0')
    const random = Math.floor(Math.random() * 10000).toString().padStart(4, '0')
    inpatientNo = `ZY${dateStr}${random}`
    
    const existing = await InpatientPatient.findOne({
      where: { inpatientNo },
    })
    
    exists = !!existing
  }
  
  return inpatientNo
}
```

## 注意事项

### 1. 性能考虑

**查询开销**：
- 每次创建患者时最多进行2次查询
  1. 检查住院号是否存在
  2. 生成唯一住院号时的循环检查（通常1次）

**优化建议**：
```typescript
// 如果住院号冲突频繁，可以考虑添加缓存
const cache = new Map<string, boolean>()

async function isInpatientNoUsed(inpatientNo: string): Promise<boolean> {
  if (cache.has(inpatientNo)) {
    return cache.get(inpatientNo)!
  }
  
  const existing = await InpatientPatient.findOne({
    where: { inpatientNo },
    attributes: ['id'],
  })
  
  const used = !!existing
  cache.set(inpatientNo, used)
  
  // 5分钟后清除缓存
  setTimeout(() => cache.delete(inpatientNo), 5 * 60 * 1000)
  
  return used
}
```

const requestedNos = patients.map(p => p.inpatientNo).filter(Boolean)
const existingNos = await InpatientPatient.findAll({

const existingSet = new Set(existingNos.map(n => n.inpatientNo))
