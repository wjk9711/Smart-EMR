# 病案首页质控功能测试指南

## 🧪 测试步骤

### 1. 启动开发环境

```bash
# 后端（如果需要）
cd emr-backend
npm run dev

# 前端
cd emr-frontend
npm run dev
```

### 2. 访问病案首页

```
1. 浏览器打开 http://localhost:5173（或实际端口）
2. 登录系统
3. 进入"住院管理" → "住院患者"
4. 选择一个患者（如：小丽）
5. 点击"病案列表"
6. 点击"新增病案" → "病案首页"
```

## ✅ 测试用例

### 测试组1：出院诊断质控

#### 测试1.1：编码正确
**操作步骤**：
1. 在出院诊断表格中，点击"+ 添加诊断"
2. 填写诊断名称：`肺炎`
3. 填写疾病编码：`J18.9`
4. 选择入院病情：`1.有`
5. 点击该行的"质控"按钮

**预期结果**：
- ✅ 显示绿色成功消息："编码正确"

---

#### 测试1.2：编码不匹配

## 🔍 常见问题排查

### 问题1：点击质控按钮无反应

**可能原因**：
- JavaScript错误
- 函数未正确绑定

**排查方法**：
1. 打开浏览器开发者工具（F12）
2. 查看Console标签是否有错误
3. 检查Network标签是否有失败的请求

---

### 问题2：建议编码不准确

**可能原因**：
- 编码数据库中缺少该疾病/手术
- 名称拼写不完全匹配

**解决方法**：
1. 检查 `homePageQualityControl.ts` 中的编码数据库
2. 添加缺失的编码映射
3. 或使用更精确的疾病/手术名称

---

### 问题3：确认框不弹出

**可能原因**：
- ElMessageBox未正确导入
- 代码逻辑有误

**排查方法**：
1. 检查HomePageForm.vue的import语句
2. 确认包含：`import { ElMessage, ElMessageBox } from 'element-plus'`

---

### 问题4：编码验证失败但应该通过

**可能原因**：
- 编码格式验证正则表达式过于严格
- 编码数据库中的编码格式不一致

**解决方法**：
1. 检查 `validateICD10CodeFormat` 和 `validateICD9CM3CodeFormat` 函数
2. 调整正则表达式以支持更多合法格式
