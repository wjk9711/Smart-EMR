# 病案首页编辑时患者基本信息不显示问题修复报告

## 📅 修复日期
2026-06-08

## 🐛 问题描述

**用户反馈**：在点击编辑病案时，患者的基本信息（姓名、性别等元素）没有带入

**问题现象**：
1. 新增病案首页时，患者信息自动填充正常
2. 保存病案后关闭弹窗
3. 再次点击"编辑"该病案
4. 患者基本信息字段（姓名、性别、年龄等）显示为空
5. 但出院诊断和手术操作能正确显示

---

## 🔍 问题分析

### 根本原因

**数据覆盖问题**：`loadRecordData` 函数覆盖了从患者表加载的基本信息

#### 执行流程

```
打开编辑弹窗
  ↓
watch 触发
  ↓
1. loadPatientInfo() - 从患者表加载基本信息
   - name: "张三"
   - gender: "1"
   - age: "45"
   - ...
  ↓
2. loadRecordData() - 从病案记录加载数据
   - 遍历 homePage 所有字段
   - 如果字段存在（即使是空字符串），就覆盖 formData
   - homePage.name = "" → 覆盖了 "张三" ❌
   - homePage.gender = "" → 覆盖了 "1" ❌
   - homePage.age = "" → 覆盖了 "45" ❌
  ↓
结果：患者基本信息被清空！
```

---

### 代码问题定位

#### 问题代码

```typescript
// ❌ 原有代码（有问题）
const loadRecordData = () => {
  if (props.recordData?.content) {
    const content = JSON.parse(props.recordData.content)
    const homePage = content.homePage || {}
    
    // 填充所有字段（包括患者基本信息）
    Object.keys(formData).forEach(key => {
      if (homePage[key] !== undefined) {
        (formData as any)[key] = homePage[key]  // ❌ 会覆盖患者信息
      }
    })
  }
}
```

**问题**：
1. 遍历 `formData` 的所有键
2. 如果 `homePage` 中有对应的键（即使是空字符串 `""`）
3. 就会覆盖之前从患者表加载的数据
4. 导致患者基本信息丢失

---

### 为什么会这样？

#### 数据存储结构

**保存时的数据结构**：
```json
{
  "homePage": {
    "name": "",           // 空字符串（不应该保存）
    "gender": "",         // 空字符串（不应该保存）
    "age": "",            // 空字符串（不应该保存）
    "medicalPaymentMethod": "1",
    "dischargeDiagnoses": [...],
    "operations": [...]
  }
}
```

**问题**：
- 患者基本信息字段也被保存到病案中（虽然是空值）
- 加载时，这些空值覆盖了从患者表加载的正确数据

---

## ✅ 修复方案

### 核心思路

**保护患者基本信息字段不被覆盖**

1. 定义不应该被覆盖的字段列表
2. 在加载病案数据时，跳过这些字段
3. 只加载病案特有的字段

---

### 修复代码

```typescript
// ✅ 修复后代码
const loadRecordData = () => {
  if (props.recordData?.content) {
    try {
      const content = typeof props.recordData.content === 'string' 
        ? JSON.parse(props.recordData.content) 
        : props.recordData.content
      
      const homePage = content.homePage || {}
      
      // 定义不应该被病案数据覆盖的患者基本信息字段
      const patientInfoFields = [
        'name', 'gender', 'birthDate', 'age', 'idCardNo', 'phone', 'presentAddress',
        'inpatientNo', 'caseNo'
      ]
      
      // 填充病案特有字段（排除患者基本信息）
      Object.keys(homePage).forEach(key => {
        // 跳过患者基本信息字段，保留从患者表加载的数据
        if (!patientInfoFields.includes(key) && homePage[key] !== undefined) {
          (formData as any)[key] = homePage[key]
        }
      })
      
      console.log('病案数据已加载，患者基本信息保持不变')
      
      // 同步诊断和手术列表
      if (homePage.dischargeDiagnoses && Array.isArray(homePage.dischargeDiagnoses)) {
        dischargeDiagnoses.value = homePage.dischargeDiagnoses
      } else {
        dischargeDiagnoses.value = []
      }
      
      if (homePage.operations && Array.isArray(homePage.operations)) {
        operations.value = homePage.operations
      } else {
        operations.value = []
      }
    } catch (error) {
      console.error('Failed to parse record content:', error)
    }
  }
}
```

---

### 关键改进

#### 1. 定义保护字段列表

```typescript
const patientInfoFields = [
  'name', 'gender', 'birthDate', 'age', 'idCardNo', 'phone', 'presentAddress',
  'inpatientNo', 'caseNo'
]
```

**包含的字段**：
- **患者身份信息**：姓名、性别、出生日期、年龄
- **联系方式**：身份证号、电话、现住址
- **病案标识**：住院号、病案号

---

#### 2. 反向遍历逻辑

```typescript
// ❌ 修复前：遍历 formData 的键
Object.keys(formData).forEach(key => {
  if (homePage[key] !== undefined) {
    (formData as any)[key] = homePage[key]
  }
})

// ✅ 修复后：遍历 homePage 的键，并排除保护字段
Object.keys(homePage).forEach(key => {
  if (!patientInfoFields.includes(key) && homePage[key] !== undefined) {
    (formData as any)[key] = homePage[key]
  }
})
```

**优势**：
- 只处理病案中实际存在的字段
- 自动跳过患者基本信息字段
- 避免不必要的覆盖

---

#### 3. 添加调试日志

```typescript
console.log('病案数据已加载，患者基本信息保持不变')
```

**作用**：
- 确认加载顺序正确
- 便于排查问题
- 提升可维护性

## 💡 相关知识

### 数据加载顺序的重要性

```typescript
// 正确的加载顺序
watch(() => props.visible, async (val) => {
  if (val) {
    // 1. 先加载患者基本信息（基础数据）
    await loadPatientInfo(props.patientId)
    
    // 2. 再加载病案数据（业务数据）
    loadRecordData()
  }
})
```

**原则**：
- 基础数据先加载
- 业务数据后加载
- 业务数据不应覆盖基础数据

---

### 字段保护策略

#### 策略1：白名单（当前使用）

```typescript
const protectedFields = ['name', 'gender', ...]
if (!protectedFields.includes(key)) {
  // 只处理非保护字段
}
```

**优点**：
- 明确指定哪些字段需要保护
- 安全性高
- 易于维护

**缺点**：
- 需要手动维护保护字段列表

---

#### 策略2：黑名单

```typescript
const skipFields = ['createdAt', 'updatedAt', ...]
if (!skipFields.includes(key)) {
  // 处理所有字段，除了黑名单
}
```

**优点**：
- 默认处理所有字段
- 只需指定少数例外

**缺点**：
- 可能遗漏需要保护的字段
- 安全性较低

---

### Object.keys 遍历差异

```typescript
// 遍历 formData 的键
Object.keys(formData).forEach(key => {
  // 会遍历所有 formData 中定义的键
  // 即使 homePage 中没有这个键
})

// 遍历 homePage 的键
Object.keys(homePage).forEach(key => {
  // 只遍历 homePage 中实际存在的键
  // 更高效，更准确
})
```

**建议**：优先遍历源数据（homePage）的键

## ⚠️ 注意事项

### 1. 保护字段完整性

当前保护字段列表：
```typescript
const patientInfoFields = [
  'name', 'gender', 'birthDate', 'age', 'idCardNo', 'phone', 'presentAddress',
  'inpatientNo', 'caseNo'
]
```

**如果未来添加了新的患者基本字段**，需要更新此列表。

---

### 2. 数据来源优先级

**优先级规则**：
1. 患者表数据 > 病案数据（患者基本信息）
2. 病案数据 > 默认值（病案特有字段）

**实现**：
- 先加载患者表数据
- 再加载病案数据（跳过患者基本字段）

---

### 3. 向后兼容性

**问题**：旧的病案记录中可能保存了患者基本信息字段

**解决**：
- 当前修复已经处理了这种情况
- 即使旧数据中有这些字段，也不会覆盖

---

### 4. 性能考虑

- `Array.includes()` 的时间复杂度是 O(n)
- 保护字段数量少（9个），性能影响可忽略
- 如果字段很多，可以考虑使用 Set

```typescript
// 优化版本（字段很多时）
const patientInfoFields = new Set([
  'name', 'gender', 'birthDate', 'age', ...
])

if (!patientInfoFields.has(key)) {
  // ...
}
```
