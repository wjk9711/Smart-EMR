# 病案首页编辑提交服务器错误修复报告

## 📅 修复日期
2026-06-08

## 🐛 问题描述

**用户反馈**：编辑病案首页点击提交时，提示"服务器错误，提交失败"

**问题现象**：
1. 打开已保存的病案首页进行编辑
2. 修改部分内容
3. 点击"提交"按钮
4. 前端显示："服务器错误，提交失败"
5. 后端返回 500 错误

---

## 🔍 问题分析

### 根本原因

**两个关键问题**导致提交失败：

#### 问题1：TypeScript 模型定义与数据库不一致

**数据库枚举**（已扩展）：
```sql
ENUM('admission', 'progress', 'discharge', 'operation', 'home_page', 'other')
```

**TypeScript 模型**（未更新）：
```typescript
// ❌ 缺少 'home_page'
type: DataTypes.ENUM('admission', 'progress', 'discharge', 'operation', 'other')
```

**后果**：
- 当 `recordType = 'home_page'` 时
- Sequelize 验证失败
- 抛出错误：`Validation error`
- 返回 500 服务器错误

---

#### 问题2：后端缺少调试日志

**原有代码**：
```typescript
export const updateInpatientRecord = async (req: Request, res: Response) => {
  try {
    const { id } = req.params
    const record = await InpatientRecord.findByPk(id)
    
    if (!record) {
      return error(res, 404, '病案不存在')
    }
    
    // ... 处理逻辑
    
    await record.update(req.body)
    res.json({ code: 200, data: record, message: '更新成功' })
  } catch (err: any) {
    return error(res, 500, err.message)  // ❌ 没有详细日志
  }
}
```

**问题**：
- 没有请求体日志
- 没有错误详情
- 难以排查问题

---

```typescript
export const updateInpatientRecord = async (req: Request, res: Response) => {
  try {
    const { id } = req.params

    if (!record) {

    if (req.body.content && typeof req.body.content === 'object') {
      req.body.content = JSON.stringify(req.body.content)
    }
    
    // ✅ 添加处理后日志
    console.log('更新病案 - 处理后的数据:', req.body)
    
    await record.update(req.body)
    
    // ✅ 添加成功日志
    console.log('更新病案 - 成功')
    
    res.json({
      code: 200,
      data: record,
      message: '更新成功',
    })
  } catch (err: any) {
    // ✅ 添加详细错误日志
    console.error('更新病案 - 错误:', err)
    console.error('更新病案 - 错误详情:', err.message)
    return error(res, 500, err.message || '更新失败')
  }
}
```

**改进**：

## 💡 相关知识

### Sequelize 枚举验证

```typescript
// 模型定义
recordType: {
  type: DataTypes.ENUM('admission', 'progress', 'home_page'),
}

// 使用
const record = new InpatientRecord({
  recordType: 'home_page'  // ✅ 有效值
})

const record2 = new InpatientRecord({
  recordType: 'invalid'  // ❌ 验证失败，抛出错误
})
```

**重要**：
- TypeScript 模型定义必须与数据库一致
- 否则会导致验证错误

---

### 数据库迁移 vs 模型同步

#### 方式1：数据库迁移脚本（推荐）

```typescript
// extendInpatientRecord.ts
await queryInterface.sequelize.query(`
  ALTER TABLE inpatient_records 
  MODIFY COLUMN record_type ENUM('admission', 'progress', 'discharge', 'operation', 'home_page', 'other')
`)
```

**优点**：
- 直接修改数据库
- 不依赖 ORM
- 适用于生产环境

**缺点**：
- 需要手动运行脚本
- TypeScript 模型不会自动更新

---

#### 方式2：Sequelize 同步

```typescript
// 开发环境
await sequelize.sync({ alter: true })
```

**优点**：
- 自动同步模型和数据库
- 方便开发

**缺点**：
- 生产环境不推荐
- 可能导致数据丢失

---

### 最佳实践

**开发流程**：
1. 修改数据库（迁移脚本）
2. 更新 TypeScript 模型
3. 重启后端服务
4. 测试验证

**原则**：
- 数据库和模型必须保持一致
- 每次修改都要同步更新
- 添加详细的日志便于调试


**后端接收**：
```typescript

if (req.body.content && typeof req.body.content === 'object') {

```
