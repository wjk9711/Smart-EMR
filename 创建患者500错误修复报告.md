# 创建患者500错误修复报告

## 问题描述

在提交住院患者时出现500错误：
```
Executing (default): INSERT INTO `inpatient_patients` (...) VALUES (...);
POST /api/inpatient/patients 500 4.294 ms - 41
```

## 问题原因

### 根本原因
添加了`source_patient_id`字段后，Sequelize在处理INSERT语句时遇到了问题：

1. **前端发送的数据**可能包含`sourcePatientId: undefined`
2. **Sequelize序列化**时将undefined转换为SQL的undefined（而非NULL）
3. **数据库约束**期望的是NULL值或有效的外键引用
4. **结果**：INSERT失败，返回500错误

### 技术细节

```typescript
// 之前的代码（有问题）
const patientData = {
  ...req.body,  // 可能包含 sourcePatientId: undefined
  doctorId: currentUser.id,
}

// Sequelize生成的SQL
INSERT INTO inpatient_patients (..., source_patient_id) VALUES (..., undefined)
// ❌ 错误：undefined不是有效的SQL值
```

## 修复方案

### 1. 创建患者时明确处理sourcePatientId

**修改前**:
```typescript
const patientData = {
  ...req.body,
  doctorId: currentUser.id,
}
```

**修改后**:
```typescript
// 排除 sourcePatientId，新创建的患者不应该有来源
const { sourcePatientId, ...restData } = req.body
const patientData = {
  ...restData,
  doctorId: currentUser.id,
  sourcePatientId: null, // 明确设置为null
}
```

### 2. 更新患者时禁止修改sourcePatientId

**修改前**:
```typescript
await patient.update(req.body)
```

**修改后**:
```typescript
// 不允许修改 sourcePatientId
const updateData = { ...req.body }
delete updateData.sourcePatientId

await patient.update(updateData)
```

### 3. 添加错误日志

在catch块中添加详细的错误日志，便于调试：
```typescript
catch (err: any) {
  console.error('Create inpatient patient error:', err)
  return error(res, 500, err.message)
}
```

## 相关说明

### source_patient_id字段的用途

| 场景 | source_patient_id值 | 说明 |
|------|-------------------|------|
| 管理员创建原始患者 | NULL | 原始数据，没有来源 |
| 学生/教师创建患者 | NULL | 自己创建的患者 |
| 下发的副本患者 | 原始患者ID | 指向原始患者，用于追溯 |

### 为什么需要明确设置为NULL？

1. **Sequelize行为**: 当字段值为`undefined`时，Sequelize可能会尝试插入该字段
2. **数据库要求**: MySQL期望的是`NULL`关键字，而不是undefined
3. **外键约束**: `source_patient_id`有外键约束，必须是NULL或有效的患者ID
4. **最佳实践**: 显式设置可以避免歧义和潜在的错误

### 其他注意事项

#### 1. 删除患者时的级联处理

```typescript
// source_patient_id设置了 ON DELETE SET NULL
// 如果删除了原始患者，副本的source_patient_id会自动设为NULL
```

#### 2. 查询优化

```typescript
// 可以通过source_patient_id快速找到所有副本
SELECT * FROM inpatient_patients WHERE source_patient_id = 5;

// 或者找到所有原始患者（非副本）
SELECT * FROM inpatient_patients WHERE source_patient_id IS NULL;
```

#### 3. 数据完整性

- ✅ 新创建的患者：`source_patient_id = NULL`
- ✅ 下发的副本：`source_patient_id = 原始患者ID`
- ✅ 不允许手动修改`source_patient_id`（通过代码控制）
- ✅ 外键约束保证引用完整性
