# 病案首页预览宽度调整说明

## 📅 更新日期
2026-06-08

## 🎯 调整目标

**用户需求**：点击查看时显示的病案首页过宽了，在保持高度不变、布局不变的情况下，减小宽度

---

## ✅ 调整方案

### 1. 对话框宽度调整

**文件**: `emr-frontend/src/views/inpatient/HomePagePreview.vue`

#### 修改前
```vue
<el-dialog
  :model-value="visible"
  title="病案首页预览"
  width="95%"    <!-- ❌ 过宽 -->
  top="2vh"
  @update:model-value="$emit('update:visible', $event)"
>
```

#### 修改后
```vue
<el-dialog
  :model-value="visible"
  title="病案首页预览"
  width="1200px"  <!-- ✅ 固定宽度，更合适 -->
  top="2vh"
  @update:model-value="$emit('update:visible', $event)"
>
```

**改进**：
- ✅ 从百分比改为固定像素值
- ✅ 1200px 适合大多数屏幕
- ✅ 保持内容可读性
- ✅ 不会过宽导致阅读困难

---

### 2. 内容区域最大宽度限制

**文件**: `emr-frontend/src/views/inpatient/HomePagePreview.vue`

#### 修改前
```scss
.home-page-preview {
  padding: 20px;
  // 没有宽度限制
}
```

#### 修改后
```scss
.home-page-preview {
  padding: 20px;
  max-width: 1100px;  // ✅ 限制最大宽度
  margin: 0 auto;      // ✅ 居中显示
}
```

**改进**：
- ✅ 限制内容最大宽度为 1100px
- ✅ 内容自动居中
- ✅ 在大屏幕上不会过宽
- ✅ 保持布局美观

## 🎨 设计原则

### 1. 最佳阅读宽度

**研究表明**：
- 每行最佳字符数：60-80个字符
- 最佳阅读宽度：600-800px（文本）
- 表格数据：800-1200px

**我们的选择**：
- 对话框：1200px（包含边距）
- 内容区：1100px（实际内容）
- 留白：每侧50px

---

### 2. 响应式考虑

虽然使用了固定宽度，但通过以下方式保证兼容性：

```scss
.home-page-preview {
  max-width: 1100px;  // 最大宽度
  margin: 0 auto;      // 居中
}
```

**效果**：
- 大屏幕（>1200px）：内容居中，两侧留白
- 中等屏幕（1200px左右）：刚好填满
- 小屏幕（<1200px）：自动适应（Element Plus Dialog 会自动调整）

---

### 3. 布局保持不变

**未修改的部分**：
- ✅ Grid 布局：`grid-template-columns: repeat(auto-fill, minmax(250px, 1fr))`
- ✅ Flex 布局：`.info-row`
- ✅ 表格样式：`.data-table`
- ✅ 字体大小、颜色、间距
- ✅ 所有组件的相对位置

**只修改了**：
- 对话框宽度
- 内容最大宽度
- 内容居中

可以进一步减小宽度：

```vue
<!-- 选项1：更小宽度 -->
<el-dialog width="1000px">

<!-- 选项2：百分比 -->
<el-dialog width="80%">

<!-- 选项3：响应式 -->
<el-dialog :width="dialogWidth">
```

```typescript
// 响应式宽度示例
const dialogWidth = computed(() => {
  const screenWidth = window.innerWidth
  if (screenWidth < 1400) return '90%'
  if (screenWidth < 1920) return '1200px'
  return '1400px'
})
```

## 🔧 技术细节

### Element Plus Dialog 宽度属性

```typescript
interface DialogProps {
  width?: string | number  // 支持像素值或百分比
}
```

**支持的格式**：
- `'1200px'` - 固定像素
- `'80%'` - 百分比
- `1200` - 数字（等同于像素）

**默认行为**：
- 如果宽度超过屏幕，会自动调整为 `calc(100vw - 40px)`
- 始终保持至少20px的边距

---

### CSS max-width 属性

```scss
.home-page-preview {
  max-width: 1100px;  // 最大宽度
  margin: 0 auto;      // 水平居中
}
```

**工作原理**：
- 当容器宽度 < 1100px：使用容器宽度
- 当容器宽度 >= 1100px：限制为 1100px
- `margin: 0 auto`：在剩余空间中居中

## ⚠️ 注意事项

### 1. 打印功能不受影响

打印时使用独立的样式：

```typescript
const handlePrint = () => {
  const printWindow = window.open('', '_blank')
  printWindow.document.write(`
    <!DOCTYPE html>
    <html>
    <head>
      <style>
        body { 
          font-family: 'Microsoft YaHei', sans-serif; 
          padding: 20px; 
          /* 打印样式独立 */
        }
      </style>
    </head>
    <body>
      ${printContent}
    </body>
    </html>
  `)
  printWindow.print()
}
```

**结论**：预览宽度调整不影响打印效果。

---

### 2. 响应式适配

Element Plus Dialog 会自动处理小屏幕：

```typescript
// Element Plus 内部逻辑
if (dialogWidth > screenWidth - 40) {
  actualWidth = screenWidth - 40
}
```

**结论**：在小屏幕上会自动缩小，不会出现溢出。
