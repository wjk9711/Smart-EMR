# 患者下发重复病案问题修复

## ❌ 问题描述

管理员下发一个患者（包含4份病案：病案首页、入院记录、首次病程记录、出院记录）后，其他用户收到后变成了**两份**（8份病案）。

## 🔍 问题原因

**重复下发**：管理员可能点击了两次"下发患者"按钮，或者多次执行下发操作。

原代码注释写着：
```typescript
// 注意：不再检查是否已分配过，允许重复下发
// 每次下发都创建新的副本患者和病案
```

这导致每次下发都会创建新的副本，即使用户已经收到过该患者。

## ✅ 解决方案

添加重复检查逻辑，如果用户已经收到过该患者，则跳过不再创建新的副本。

### 修改文件

**文件**：`emr-backend/src/controllers/inpatientController.ts`

**修改位置**：第884-892行

**修改前**：
```typescript
for (const user of users) {
  console.log(`  → 下发给用户: ${user.id}`)
  // 注意：不再检查是否已分配过，允许重复下发
  // 每次下发都创建新的副本患者和病案
  
  // 生成新的唯一KEY...
```

**修改后**：
```typescript
for (const user of users) {
  console.log(`  → 下发给用户: ${user.id}`)
  
  // 检查是否已经下发过（避免重复创建）
  const existingAssignment = await PatientAssignment.findOne({
    where: {
      patientId: originalPatient.id,
      userId: user.id,
    },
    transaction,
  })
  
  if (existingAssignment) {
    console.log(`    ⚠️  该用户已收到此患者，跳过（分配ID: ${existingAssignment.id}）`)
    continue
  }
  
  // 生成新的唯一KEY...
```

## 🧹 清理重复数据（可选）

如果已经有重复的患者副本，可以清理：

```sql
-- 查找重复的分配记录
SELECT 
    pa.patient_id,
    pa.user_id,
    GROUP_CONCAT(pa.copied_patient_id) as copied_ids,
    COUNT(*) as count
FROM patient_assignments pa
GROUP BY pa.patient_id, pa.user_id
HAVING count > 1;

-- 删除重复的副本患者和病案（谨慎操作！）
-- 只保留最早创建的副本
DELETE p FROM inpatient_patients p
INNER JOIN (
    SELECT 
        pa1.copied_patient_id,
        ROW_NUMBER() OVER (PARTITION BY pa1.patient_id, pa1.user_id ORDER BY pa1.copied_at) as rn
    FROM patient_assignments pa1
) ranked ON p.id = ranked.copied_patient_id
WHERE ranked.rn > 1;

-- 删除重复的分配记录
DELETE pa FROM patient_assignments pa
INNER JOIN (
    SELECT 
        id,
        ROW_NUMBER() OVER (PARTITION BY patient_id, user_id ORDER BY copied_at) as rn
    FROM patient_assignments
) ranked ON pa.id = ranked.id
WHERE ranked.rn > 1;
```
