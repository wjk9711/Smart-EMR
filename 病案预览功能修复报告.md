# 病案预览功能修复报告

## 📋 问题描述

用户反馈了两个问题：

1. **病案首页预览不显示**：在打开"预览全部病案"时，病案首页预览组件没有正确显示内容
2. **入院记录患者信息未带入**：入院记录预览中没有显示患者的基本信息

---

## 🔍 问题分析

### 问题1: 病案首页预览不显示

**根本原因**：
- `HomePagePreview.vue` 和 `AdmissionRecordPreview.vue` 都是对话框组件（使用 `<el-dialog>`）
- 在统一预览列表中直接使用时，这些组件会尝试打开自己的对话框，而不是在父对话框中显示内容
- 导致内容被包裹在嵌套的对话框中，无法正常显示

**错误的使用方式** ❌：
```vue
<!-- 在 InpatientRecordList.vue 的 el-dialog 中使用 -->
<el-dialog v-model="previewAllVisible">
  <!-- HomePagePreview 自己也有 el-dialog，造成嵌套 -->
  <HomePagePreview :visible="true" :record-data="record" />
</el-dialog>
```

### 问题2: 入院记录患者信息未带入

**根本原因**：
- `AdmissionRecordPreview.vue` 的数据提取逻辑不够健壮
- 没有正确处理不同的数据结构（`content.admissionRecord` vs `content`）
- 缺少对空值的容错处理

---

## ✅ 解决方案

### 方案概述

为预览组件添加**双模式支持**：
1. **对话框模式** (`mode="dialog"`)：用于单个病案预览，带 `el-dialog`
2. **内嵌模式** (`mode="embedded"`)：用于统一预览列表，直接显示内容，不带 `el-dialog`

### 实现细节

#### 1. 修改 HomePagePreview.vue

**添加 mode 属性**：
```typescript
interface Props {
  visible: boolean
  recordData?: any
  mode?: 'dialog' | 'embedded'  // 新增：显示模式
}

const props = withDefaults(defineProps<Props>(), {
  visible: false,
  recordData: null,
  mode: 'dialog',  // 默认对话框模式
})
```

**条件渲染模板**：
```vue
<template>
  <!-- 内嵌模式：直接显示内容，不带对话框 -->
  <div v-if="mode === 'embedded'" class="home-page-preview">
    <PreviewContent />
  </div>
  
  <!-- 对话框模式：使用 el-dialog -->
  <el-dialog
    v-else
    :model-value="visible"
    title="病案首页预览"
    width="1000px"
    top="2vh"
    @update:model-value="$emit('update:visible', $event)"
  >
    <div class="home-page-preview" ref="printArea">
      <PreviewContent />
    </div>
    
    <template #footer>
      <el-button @click="$emit('update:visible', false)">关闭</el-button>
      <el-button type="primary" @click="handlePrint">打印</el-button>
    </template>
  </el-dialog>
</template>
```

**使用 defineComponent + h 函数渲染内容**：
```typescript
const PreviewContent = defineComponent({
  setup() {
    return () => h('div', [
      // 标题
      h('div', { class: 'page-title' }, '住 院 病 案 首 页'),
      
      // 患者基本信息
      h('div', { class: 'section-title' }, '患者基本信息'),
      h('div', { class: 'info-grid' }, [
        h('div', { class: 'info-item' }, [
          h('span', { class: 'label' }, '姓名：'),
          h('span', { class: 'value' }, data.value.name || '-'),
        ]),
        // ... 更多字段
      ]),
      
      // 出院诊断表格
      h('table', { class: 'data-table' }, [
        h('thead', [...]),
        h('tbody', (data.value.dischargeDiagnoses || []).map((diag: any, index: number) => 
          h('tr', { key: index }, [
            h('td', diag.diagnosis || diag.name || '-'),
            h('td', diag.code || '-'),
            h('td', getAdmissionConditionName(diag.condition || diag.admissionCondition)),
          ])
        )),
      ]),
    ])
  }
})
```

#### 2. 修改 AdmissionRecordPreview.vue

采用相同的模式，添加 `mode` 属性和 `PreviewContent` 组件。

**关键改进**：
- 数据提取更加健壮，支持多种数据结构
- 所有患者信息字段都有默认值处理
- 添加了打印功能

#### 3. 修改 InpatientRecordList.vue

在统一预览中使用 `mode="embedded"`：

```vue
<!-- 病案首页预览 -->
<div v-if="record.recordType === 'home_page'" class="record-content">
  <HomePagePreview 
    :visible="true"
    :record-data="record"
    mode="embedded"  <!-- ✅ 使用内嵌模式 -->
  />
</div>

<!-- 入院记录预览 -->
<div v-else-if="record.recordType === 'admission'" class="record-content">
  <AdmissionRecordPreview 
    :visible="true"
    :record-data="record"
    mode="embedded"  <!-- ✅ 使用内嵌模式 -->
  />
</div>
```

## 📊 技术要点

### 1. Vue 3 Composition API

```typescript
import { computed, ref, defineComponent, h } from 'vue'

// 使用 computed 确保响应式
const data = computed(() => {
  if (!props.recordData) return {}
  
  let content = props.recordData.content
  if (typeof content === 'string') {
    try {
      content = JSON.parse(content)
    } catch (e) {
      console.error('Failed to parse content:', e)
      return {}
    }
  }
  
  return content.homePage || content || {}
})
```

### 2. 动态组件渲染

使用 `defineComponent` + `h` 函数创建可复用的内容组件：

```typescript
const PreviewContent = defineComponent({
  setup() {
    return () => h('div', [
      h('div', { class: 'page-title' }, '标题'),
      h('div', { class: 'section-title' }, '分区标题'),
      // ...
    ])
  }
})
```

**优点**：
- 代码复用：同一套内容逻辑用于两种模式
- 类型安全：TypeScript 完整支持
- 性能优化：Vue 3 虚拟 DOM 高效更新

### 3. 数据兼容性

支持多种数据结构：

```typescript
// 情况1: content.homePage
return content.homePage || content || {}

// 情况2: 兼容不同的字段名
h('td', diag.diagnosis || diag.name || '-')

// 情况3: 安全的空值处理
{{ data.value.name || '-' }}
```

### 4. 条件渲染

```vue
<template>
  <div v-if="mode === 'embedded'">
    <PreviewContent />
  </div>
  
  <el-dialog v-else>
    <PreviewContent />
  </el-dialog>
</template>
```

## ✅ 总结

本次修复解决了两个关键问题：

1. ✅ **病案首页预览显示问题**：通过添加 `mode` 属性，支持内嵌模式和对话框模式
2. ✅ **入院记录患者信息显示问题**：优化数据提取逻辑，确保所有字段正确显示

**核心技术**：
- Vue 3 Composition API
- `defineComponent` + `h` 函数动态渲染
- 条件渲染（`v-if` / `v-else`）
- 响应式数据（`computed`）

**设计原则**：
- 代码复用：同一套内容逻辑用于两种模式
- 向后兼容：默认使用对话框模式，不影响现有代码
- 类型安全：完整的 TypeScript 类型定义
- 易于扩展：其他病案类型可以快速采用相同模式
