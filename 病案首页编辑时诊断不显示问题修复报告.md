# 病案首页编辑时出院诊断不显示问题修复报告

## 📅 修复日期
2026-06-08

## 🐛 问题描述

**用户反馈**：增加的出院诊断在点击编辑时并没有显示

**问题现象**：
1. 用户新增病案首页，添加了多个出院诊断
2. 保存后关闭弹窗
3. 再次点击编辑该病案
4. 出院诊断列表为空，之前添加的诊断没有显示

---

## 🔍 问题分析

### 根本原因

**加载顺序问题** + **数据验证缺失**

#### 问题1：异步加载顺序错误

```typescript
// ❌ 原有代码（错误）
watch(() => props.visible, (val) => {
  if (val) {
    if (props.patientId) {
      loadPatientInfo(props.patientId)  // 异步函数，但没有await
    }
    if (props.recordData) {
      loadRecordData()  // 同步执行，可能在loadPatientInfo之前完成
    }
  }
})
```

**问题**：
- `loadPatientInfo` 是异步函数，但没有使用 `await`
- `loadRecordData` 可能先于 `loadPatientInfo` 执行完成
- 导致数据加载顺序不确定

---

#### 问题2：数据验证不足

```typescript
// ❌ 原有代码（不够健壮）
dischargeDiagnoses.value = homePage.dischargeDiagnoses || []
operations.value = homePage.operations || []
```

**问题**：
- 没有验证数据类型是否为数组
- 如果 `homePage.dischargeDiagnoses` 是 `null`、`undefined` 或非数组类型，会导致错误
- 缺少调试日志，难以排查问题

---

## ✅ 修复方案

### 修复1：确保加载顺序正确

```typescript
// ✅ 修复后代码
watch(() => props.visible, async (val) => {
  if (val) {
    // 先加载患者基本信息
    if (props.patientId) {
      await loadPatientInfo(props.patientId)  // 等待完成
    }
    
    // 再加载病案数据（会覆盖部分患者信息）
    if (props.recordData) {
      loadRecordData()
    } else {
      // 新增模式，初始化默认值
      dischargeDiagnoses.value = []
      operations.value = []
    }
  }
})
```

**改进**：
- ✅ 使用 `async/await` 确保 `loadPatientInfo` 完成后才执行后续代码
- ✅ 明确的注释说明加载顺序
- ✅ 保证数据加载的确定性

---

### 修复2：增强数据验证和调试

```typescript
// ✅ 修复后代码
const loadRecordData = () => {
  if (props.recordData?.content) {
    try {
      const content = typeof props.recordData.content === 'string' 
        ? JSON.parse(props.recordData.content) 
        : props.recordData.content
      
      console.log('加载病案数据 - 原始content:', content)
      
      const homePage = content.homePage || {}
      
      console.log('加载病案数据 - homePage:', homePage)
      console.log('加载病案数据 - dischargeDiagnoses:', homePage.dischargeDiagnoses)
      console.log('加载病案数据 - operations:', homePage.operations)
      
      // 填充所有字段
      Object.keys(formData).forEach(key => {
        if (homePage[key] !== undefined) {
          (formData as any)[key] = homePage[key]
        }
      })
      
      // 同步诊断和手术列表（确保是数组）
      if (homePage.dischargeDiagnoses && Array.isArray(homePage.dischargeDiagnoses)) {
        dischargeDiagnoses.value = homePage.dischargeDiagnoses
        console.log('已加载出院诊断:', dischargeDiagnoses.value.length, '条')
      } else {
        dischargeDiagnoses.value = []
        console.log('未找到出院诊断数据，初始化为空数组')
      }
      
      if (homePage.operations && Array.isArray(homePage.operations)) {
        operations.value = homePage.operations
        console.log('已加载手术操作:', operations.value.length, '条')
      } else {
        operations.value = []
        console.log('未找到手术操作数据，初始化为空数组')
      }
    } catch (error) {
      console.error('Failed to parse record content:', error)
    }
  } else {
    console.log('没有找到recordData或content')
  }
}
```

**改进**：
- ✅ 添加详细的调试日志，便于排查问题
- ✅ 验证数据类型是否为数组（`Array.isArray`）
- ✅ 明确的处理分支（有数据 vs 无数据）
- ✅ 记录加载的数据条数

## 💡 相关知识

### Vue 3 响应式系统

```typescript
// ref 创建的数组
const dischargeDiagnoses = ref([])

// 正确的赋值方式
dischargeDiagnoses.value = newArray  // ✅ 触发响应式更新

// 错误的赋值方式
dischargeDiagnoses = newArray  // ❌ 不会触发更新
```

### 异步加载顺序

```typescript
// ❌ 错误：不等待异步操作
watch(() => props.visible, (val) => {
  loadAsync1()  // 异步
  loadSync2()   // 同步，可能先完成
})

// ✅ 正确：等待异步操作完成
watch(() => props.visible, async (val) => {
  await loadAsync1()  // 等待完成
  loadSync2()         // 确定在之后执行
})
```

### 数组类型验证

```typescript
// ❌ 不安全的判断
if (data) { ... }  // 无法区分数组和其他类型

// ✅ 安全的判断
if (data && Array.isArray(data)) { ... }  // 确保是数组
```
