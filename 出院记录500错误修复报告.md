# 出院记录500错误修复报告

## 📋 问题描述

用户反馈：出院记录提交时提示500错误
```
POST /api/inpatient/records 500 16.398 ms - 41
```

同时需要检查并修改预览出院记录功能。

---

## 🔍 问题分析

### 根本原因

所有病案表单在**新建病案**时都手动传递了 `caseNo` 字段，但是：

1. **前端传递的 caseNo 可能为空或不正确**
   - 出院记录：`caseNo: formData.caseNo`（新建时为空字符串）
   - 入院记录：同样的问题
   - 首次病程记录：同样的问题
   - 病案首页：同样的问题

2. **后端已有自动设置 caseNo 的逻辑**
   ```typescript
   // 病案号统一使用患者的住院号（inpatientNo）
   if (req.body.patientId) {
     const patient = await InpatientPatient.findByPk(req.body.patientId)
     if (patient && patient.inpatientNo) {
       req.body.caseNo = patient.inpatientNo
     }
   }
   ```

3. **冲突导致的问题**
   - 前端传递空字符串或错误的 caseNo
   - 后端尝试设置为患者住院号
   - 可能导致唯一约束冲突或数据不一致

---

## ✅ 解决方案

### 方案：新建时不传递 caseNo，由后端自动设置

**原则**：
- **新建病案**：不传递 `caseNo`，由后端从患者表获取住院号并设置
- **编辑病案**：传递 `caseNo`，保持原有的病案号不变

### 1. 统一的数据流

- 打印功能

- 预览组件遵循统一的设计模式
