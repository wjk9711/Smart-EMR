# 患者下发功能修复 - 快速部署指南

## 📋 修复概述

本次修复确保下发患者时：
- ✅ **只重新生成**：`unique_key`（13位唯一标识）和 `inpatient_no`（住院号）
- ✅ **保持原样**：其他所有患者信息（姓名、身份证号、电话等）

## 📊 修改内容总结

### 模型层 (`InpatientPatient.ts`)

```typescript
// 只有 unique_key 保留唯一约束
uniqueKey: {
  type: DataTypes.STRING(13),
  allowNull: true,
  unique: true,  // ✅ 保留
  comment: '患者唯一标识（13位随机字符串）',
}

// 其他字段都不设置 unique
idCard: { ... },      // ❌ 无 unique
phone: { ... },       // ❌ 无 unique
inpatientNo: { ... }, // ❌ 无 unique
name: { ... },        // ❌ 无 unique
// ... 所有其他字段
```

### 控制器层 (`inpatientController.ts`)

```typescript
// 下发患者时只重新生成两个字段
const newInpatientNo = await generateUniqueInpatientNo()  // ✅ 新生成
const newUniqueKey = await generateUniqueKey('inpatient_patients')  // ✅ 新生成

// 其他所有字段保持原样
const copiedPatient = await InpatientPatient.create({
  name: originalPatient.name,           // ✅ 原样
  idCard: originalPatient.idCard,       // ✅ 原样
  phone: originalPatient.phone,         // ✅ 原样
  // ... 其他字段都原样复制
  inpatientNo: newInpatientNo,          // ✅ 新生成
  uniqueKey: newUniqueKey,              // ✅ 新生成
  doctorId: user.id,                    // ✅ 设置为接收者
})
```

### 数据库层

运行迁移脚本后，`inpatient_patients` 表中：
- ✅ 保留 `id` (主键) 的唯一性
- ✅ 保留 `unique_key` 的唯一性
- ❌ 移除所有其他字段的唯一索引

## ⚠️ 注意事项

1. **数据备份**：修复前已自动备份代码文件
2. **数据库影响**：迁移脚本会修改数据库结构，但不会影响现有数据
3. **回滚方案**：如需回滚，恢复备份文件并重启服务即可
4. **监控日志**：部署后密切监控 PM2 日志至少 5 分钟
