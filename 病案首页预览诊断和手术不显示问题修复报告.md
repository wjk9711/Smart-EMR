# 病案首页预览诊断和手术不显示问题修复报告

## 📅 修复日期
2026-06-08

## 🐛 问题描述

**用户反馈**：在病案首页预览中，已添加的出院诊断以及手术及操作没有同步显示

**问题现象**：
1. 新增/编辑病案首页时，添加了多个出院诊断和手术操作
2. 保存病案后关闭弹窗
3. 点击"查看"按钮打开预览
4. 患者基本信息、其他字段都能正常显示
5. **但出院诊断表格和手术操作表格显示为空**（或显示"暂无诊断信息"/"暂无手术信息"）

---

## 🔍 问题分析

### 根本原因

**数据加载时机问题**：预览组件的 watch 监听器配置不正确，导致数据没有在正确时机加载。

#### 原有代码（有问题）

```typescript
// ❌ 只监听 recordData 变化
watch(() => props.recordData, () => {
  if (props.visible) {
    loadData()
  }
}, { immediate: true })
```

#### 问题场景分析

**场景1：首次打开预览**
```
用户点击"查看"按钮
  ↓
props.visible = true
props.recordData = {...} (已有数据)
  ↓
watch 触发（immediate: true）
  ↓
loadData() 执行 ✅
  ↓
数据正常显示
```

**场景2：关闭后再次打开预览（问题场景）**
```
第一次查看后关闭弹窗
  ↓
props.visible = false
props.recordData = {...} (保持不变)
  ↓
用户再次点击"查看"同一病案
  ↓
props.visible = true
props.recordData = {...} (还是同一个对象引用，未变化)
  ↓
watch 不触发（因为 recordData 没变）❌
  ↓
loadData() 不执行
  ↓
data.value 保持旧值或空值
  ↓
出院诊断和手术操作不显示 ❌
```

**场景3：查看不同病案**
```
从病案A切换到病案B
  ↓
props.recordData 发生变化
  ↓
watch 触发 ✅
  ↓
loadData() 执行
  ↓
数据正常显示
```

---

## ✅ 修复方案

### 修复策略

**同时监听两个状态**：
1. `props.visible` - 弹窗打开时加载数据
2. `props.recordData` - 数据变化时重新加载

### 修复代码

**文件**: `emr-frontend/src/views/inpatient/HomePagePreview.vue`

```typescript
// ✅ 修复后代码

// 监听弹窗打开事件
watch(() => props.visible, (val) => {
  if (val && props.recordData) {
    console.log('预览组件 - 弹窗打开，加载数据')
    loadData()
  }
})

// 也监听 recordData 的变化
watch(() => props.recordData, () => {
  if (props.visible) {
    console.log('预览组件 - recordData 变化，重新加载数据')
    loadData()
  }
}, { immediate: true })
```

### 增强调试日志

在 `loadData` 函数中添加详细日志，便于排查问题：

```typescript
const loadData = () => {
  if (props.recordData?.content) {
    try {
      const content = typeof props.recordData.content === 'string' 
        ? JSON.parse(props.recordData.content) 
        : props.recordData.content
      
      console.log('预览组件 - 原始content:', content)
      console.log('预览组件 - homePage:', content.homePage)
      console.log('预览组件 - dischargeDiagnoses:', content.homePage?.dischargeDiagnoses)
      console.log('预览组件 - operations:', content.homePage?.operations)
      
      data.value = content.homePage || {}
    } catch (error) {
      console.error('Failed to parse record content:', error)
      data.value = {}
    }
  }
}
```

## 💡 相关知识

### Vue Watch 机制

#### immediate 选项

```typescript
watch(source, callback, { immediate: true })
```

- `immediate: true`：立即执行一次回调（组件挂载时）
- 只在 source 初次绑定时触发
- 后续只有在 source 变化时才触发

#### 监听对象引用

```typescript
watch(() => props.recordData, callback)
```

- 监听的是**对象引用**，不是对象内容
- 如果对象引用不变（即使内部属性变化），watch 不会触发
- 这是 JavaScript 的对象引用特性

### 最佳实践

**监听多个相关状态**：

```typescript
// ✅ 推荐：分别监听不同状态
watch(() => props.visible, handleVisibleChange)
watch(() => props.data, handleDataChange)

// ❌ 不推荐：试图用一个 watch 监听所有
watch([() => props.visible, () => props.data], callback)
```

**添加调试日志**：

```typescript
watch(() => props.visible, (val) => {
  console.log('visible 变化:', val)
  if (val) {
    loadData()
  }
})
```

## ⚠️ 注意事项

### 1. 数据结构确认

确保保存的数据结构正确：

```javascript
{
  homePage: {
    name: "张三",
    gender: "1",
    // ... 其他字段
    dischargeDiagnoses: [
      { name: "高血压", code: "I10", admissionCondition: "1" },
      { name: "糖尿病", code: "E11", admissionCondition: "1" }
    ],
    operations: [
      { name: "阑尾切除术", date: "2026-06-01", level: "二级" }
    ]
  }
}
```

### 2. Console 日志

修复后，打开浏览器 Console（F12），应该能看到以下日志：

```
预览组件 - 弹窗打开，加载数据
预览组件 - 原始content: {homePage: {...}}
预览组件 - homePage: {name: "张三", ...}
预览组件 - dischargeDiagnoses: [{...}, {...}]
预览组件 - operations: [{...}]
```

如果看不到这些日志，说明：
- 弹窗没有正确打开
- recordData 为空
- content 字段不存在

### 3. 性能考虑

现在的实现会在每次弹窗打开时重新加载数据，对于大数据量可能会有轻微性能影响。如果未来需要优化，可以考虑：

```typescript
// 缓存机制
const cache = new Map()

watch(() => props.visible, (val) => {
  if (val && props.recordData) {
    const cacheKey = props.recordData.id
    if (cache.has(cacheKey)) {
      data.value = cache.get(cacheKey)
    } else {
      loadData()
      cache.set(cacheKey, data.value)
    }
  }
})
```

## ✅ 总结

### 问题根源
- watch 只监听了 `recordData`，没有监听 `visible`
- 当弹窗重新打开但 `recordData` 未变化时，watch 不触发
- 导致数据不加载，诊断和手术信息不显示

### 解决方案
- 添加 `props.visible` 监听器
- 弹窗打开时强制加载数据
- 保留 `props.recordData` 监听器处理数据切换

### 改进效果
- ✅ 首次打开预览：正常显示
- ✅ 关闭后再次打开：**正常显示**（之前有问题）
- ✅ 切换不同病案：正常显示
- ✅ 编辑后查看：显示最新数据
