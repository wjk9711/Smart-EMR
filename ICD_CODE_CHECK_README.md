# ICD-10编码检查功能说明

## 功能概述

在病案表单的诊断字段中，集成了通义千问AI助手，可以智能检查诊断与ICD-10编码的对应关系，并提供编码推荐。

## 界面布局

每个诊断字段包含三个部分（横向排列）：
1. **诊断输入框**（占50%宽度）- 输入诊断名称
2. **编码输入框**（占33%宽度）- 输入ICD-10编码
3. **检查按钮**（占17%宽度）- 点击检查编码正确性

## 使用方法

### 1. 输入诊断和编码
```
入院诊断：社区获得性肺炎
编码：J18.9
```

### 2. 点击"检查编码"按钮
- 系统会调用通义千问API进行检查
- 按钮显示加载状态

### 3. 查看检查结果

#### 情况A：编码正确
```
✓ 编码正确！J18.9 对应 社区获得性肺炎
```

#### 情况B：编码不正确或未填写
弹出对话框显示：
```
编码可能不正确

当前编码：J18.0
诊断：社区获得性肺炎

推荐编码：
J18.9 - 肺炎，未特指
J18.0 - 支气管肺炎，未特指
J18.1 - 肺段肺炎，未特指
```

点击"使用推荐编码"后，自动填充第一个推荐编码。

## API配置

### 1. 获取通义千问API Key
访问阿里云DashScope控制台：https://dashscope.console.aliyun.com/

### 2. 配置环境变量
在 `.env` 文件中添加：
```env
QWEN_API_KEY=your-qwen-api-key-here
```

### 3. 重启后端服务
```bash
npm run dev
```

## 技术实现

### 前端
- **文件**: `emr-frontend/src/views/inpatient/InpatientRecordForm.vue`
- **功能**: 
  - 三列布局（诊断、编码、检查按钮）
  - 调用后端API `/api/inpatient/check-icd-code`
  - 显示检查结果和推荐编码
  - 支持自动填充推荐编码

### 后端
- **文件**: `emr-backend/src/controllers/icdCodeController.ts`
- **路由**: `POST /api/inpatient/check-icd-code`
- **功能**:
  - 接收诊断和当前编码
  - 调用通义千问API进行智能分析
  - 解析AI响应并返回结构化结果
  - 支持模拟数据（未配置API Key时）

## AI提示词设计

系统使用精心设计的提示词，要求AI：
1. 判断编码是否正确
2. 提供最多3个推荐编码
3. 每个编码包含详细描述
4. 以JSON格式返回结果

## 错误处理

- **未输入诊断**: 提示"请先输入诊断名称"
- **API调用失败**: 显示具体错误信息
- **JSON解析失败**: 返回默认错误结果
- **未配置API Key**: 使用模拟数据进行测试

## 示例场景

### 场景1：正确编码
```
诊断：急性心肌梗死
编码：I21.9
结果：✓ 编码正确！
```

### 场景2：错误编码
```
诊断：高血压
编码：E11.9（糖尿病编码）
结果：✗ 编码不正确
推荐：I10 - 原发性高血压
```

### 场景3：未填写编码
```
诊断：2型糖尿病
编码：（空）
结果：✗ 未填写编码
推荐：E11.9 - 2型糖尿病
```

## 注意事项

1. **API费用**: 通义千问API按调用次数计费，请合理使用
2. **网络要求**: 需要能够访问阿里云API服务器
3. **响应时间**: AI分析通常需要1-3秒
4. **准确性**: AI推荐仅供参考，最终编码需医生确认
5. **隐私保护**: 诊断数据会发送到阿里云服务器，请注意患者隐私
