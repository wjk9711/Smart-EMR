# 单胎活产柱形图显示问题修复

## 🐛 问题描述

**用户反馈**：单胎活产的柱形图没有显示。

---

## 🔍 问题分析

### 根本原因

Y轴的最小值（`min`）设置为80，而单胎活产的正确率是40.8%，低于最小值，导致柱形图无法在图表中显示。

```typescript
// ❌ 问题代码
yAxis: {
  type: 'value',
  min: 80,  // 最小值80%
  max: 100,
}

// 单胎活产正确率：40.8% < 80% → 不显示
```

### 为什么之前设置为80？

之前的诊断数据中，所有正确率都在87%-95%之间，设置 `min: 80` 可以：
- ✅ 放大差异，让柱形图高度变化更明显
- ✅ 避免底部空白过多
- ✅ 聚焦于高正确率区间

但是，添加单胎活产（40.8%）后，这个设置就不适用了。

---

## ✅ 修复方案

### 修改Y轴最小值

**文件**: [QualityControlEvaluation.vue](file://f:/File/Project/EMR/emr-frontend/src/views/quality/QualityControlEvaluation.vue#L240-L252)

**位置**: 第242行

```typescript
// 修改前
yAxis: {
  type: 'value',
  min: 80,  // ❌ 导致40.8%的数据不显示
  max: 100,
}

// 修改后
yAxis: {
  type: 'value',
  min: 0,   // ✅ 确保所有数据都能显示
  max: 100,
}
```

## 🎨 视觉优化建议

### 当前效果

- **单胎活产**：红色渐变 (#f56c6c → #f78989)，高度约40%
- **其他诊断**：绿色或橙色渐变，高度80%-95%
- **平均值线**：红色虚线在87.1%位置

### 可能的进一步优化

如果需要更好的视觉效果，可以考虑：

#### 方案1：保持min=0（当前方案）✅

**优点**：
- 所有数据完整显示
- 真实反映数据分布
- 单胎活产问题一目了然

**缺点**：
- 高正确率诊断的柱形图都很高，差异不明显
- 底部有大量空白（0-40%区间）

#### 方案2：使用双Y轴

```typescript
yAxis: [
  {
    type: 'value',
    name: '常规诊断',
    min: 80,
    max: 100,
    position: 'left',
  },
  {
    type: 'value',
    name: '异常诊断',
    min: 0,
    max: 50,
    position: 'right',
  },
]
```

**优点**：
- 常规诊断差异更明显
- 异常诊断单独显示

**缺点**：
- 实现复杂
- 可能引起混淆

#### 方案3：使用断点Y轴

```typescript
yAxis: {
  type: 'value',
  axisLabel: {
    formatter: (value) => {
      if (value === 0) return '0%'
      if (value === 40) return '40%'
      if (value >= 80) return `${value}%`
      return ''
    }
  }
}
```

**优点**：
- 聚焦关键区间
- 减少空白

**缺点**：
- 可能误导用户
- 实现复杂
