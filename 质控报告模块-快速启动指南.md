## ⚠️ 常见问题

### 问题1：数据库迁移失败

**错误信息**：
```
❌ 迁移失败: Table 'quality_reports' already exists
```

**解决方案**：
表已存在，可以跳过迁移步骤，直接使用。

---

### 问题2：前端依赖安装失败

**错误信息**：
```
npm error ERESOLVE unable to resolve dependency tree
```

**解决方案**：
使用 `--legacy-peer-deps` 参数：
```bash
npm install quill docx file-saver @types/quill @types/file-saver --legacy-peer-deps
```

---

### 问题3：编辑器不显示

**可能原因**：
- Quill样式未加载
- DOM元素未就绪

**解决方案**：
检查浏览器控制台是否有错误，确保Quill CSS已正确导入。

---

### 问题4：保存Word失败

**可能原因**：
- 后端uploads目录不存在
- 权限问题

**解决方案**：
手动创建目录：
```bash
mkdir f:\File\Project\EMR\emr-backend\uploads\quality-reports
```

## 📊 数据流程

```
用户操作                    前端                      后端                  数据库
────────                   ────                      ────                  ─────

新建报告         →   POST /api/quality-reports  →  生成15位关键值    →  INSERT
                                                    创建空DOCX文件

编辑报告         →   PUT /api/quality-reports/:id →  转换HTML为DOCX  →  UPDATE
                                                    保存文件

获取列表         →   GET /api/quality-reports   →  查询用户报告     →  SELECT

删除报告         →   DELETE /api/quality-reports/:id → 删除文件     →  DELETE
```

## 🎯 核心特性总结

### 1. 唯一关键值系统

- **格式**：15位大写字母+数字
- **示例**：`ABC123XYZ789DEF`
- **用途**：唯一标识每个报告，用于文件名和检索
- **保证**：数据库UNIQUE约束 + 生成时检查

### 2. Word文档存储

- **路径规则**：`uploads/quality-reports/{userId}_{reportKey}.docx`
- **格式**：标准DOCX格式，可用Microsoft Word打开
- **转换**：HTML → DOCX（使用docx库）

### 3. 权限控制

- **认证**：JWT Token
- **授权**：用户只能操作自己的报告
- **实现**：后端通过`req.user.id`过滤

### 4. 批量操作

- **多选**：表格支持checkbox多选
- **批量删除**：一次性删除多个报告及其文件
- **事务**：确保数据一致性
