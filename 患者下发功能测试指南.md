# 患者下发功能 - 快速测试指南

## 前置条件

1. ✅ 后端服务已启动（端口3000）
2. ✅ 前端服务已启动（通常端口5173）
3. ✅ 数据库迁移已完成（patient_assignments表已创建）

## 测试步骤

### 第一步：准备测试数据

#### 1. 创建测试用户
需要至少3个用户：
- **管理员账号**（roleType: 'admin'）
- **学生账号1**（roleType: 'student'）
- **教师账号1**（roleType: 'teacher'）

如果还没有这些账号，可以通过以下方式创建：
```bash
# 在 emr-backend 目录下运行
npx ts-node create-admin.ts
```

#### 2. 管理员创建测试患者
1. 使用管理员账号登录
2. 进入"住院患者列表"
3. 点击"新增患者"，创建2-3个测试患者
4. 记录患者信息（姓名、住院号等）

### 第二步：验证数据隔离（下发前）

#### 测试1：管理员视角
1. 使用管理员账号登录
2. 查看住院患者列表
3. **预期结果**：
   - ✅ 能看到自己创建的患者
   - ✅ 看不到其他用户创建的患者
   - ✅ 表格第一列有多选框
   - ✅ 顶部有"下发患者"按钮

#### 测试2：学生/教师视角（下发前）
1. 使用学生或教师账号登录
2. 查看住院患者列表
3. **预期结果**：
   - ✅ 只能看到自己创建的患者（如果没有则为空）
   - ✅ 看不到管理员创建的患者
   - ✅ 没有多选框
   - ✅ 没有"下发患者"按钮
   - ✅ 如果有模板患者，会显示"模板"标签

### 第三步：测试下发功能

#### 测试3：下发患者
1. 切换回管理员账号
2. 在患者列表中勾选1-2个患者
3. 点击"下发患者"按钮
4. 确认对话框中点击"确定"
5. **预期结果**：
   - ✅ 显示成功提示："成功将 X 个患者下发给 Y 个用户"
   - ✅ 选中的患者清空
   - ✅ 列表刷新

#### 测试4：验证下发结果（学生/教师视角）
1. 退出管理员账号
2. 使用学生或教师账号登录
3. 查看住院患者列表
4. **预期结果**：
   - ✅ 能看到自己创建的患者
   - ✅ 能看到管理员下发的患者
   - ✅ 下发的患者带有"模板"标签
   - ✅ 仍然看不到其他用户创建的患者

#### 测试5：验证病案隔离
1. 在管理员账号下，为某个患者创建病案
2. 切换到学生/教师账号
3. 查看该患者的病案列表
4. **预期结果**：
   - ✅ 能看到下发患者对应的病案
   - ✅ 不能看到其他用户的病案

### 第四步：边界测试

#### 测试6：重复下发
1. 管理员再次选择已下发的患者
2. 点击"下发患者"
3. **预期结果**：
   - ✅ 不报错
   - ✅ 提示成功（但实际不会重复插入）

#### 测试7：下发0个患者
1. 管理员不选择任何患者
2. "下发患者"按钮应为禁用状态
3. **预期结果**：
   - ✅ 按钮不可点击

#### 测试8：删除患者
1. 管理员删除一个已下发的患者
2. 切换到学生/教师账号
3. **预期结果**：
   - ✅ 该患者从列表中消失
   - ✅ patient_assignments表中的相关记录被自动删除（级联删除）

## API 测试（可选）

如果需要直接测试API，可以使用以下curl命令：

### 1. 获取患者列表
```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  "http://localhost:3000/api/inpatient/patients?page=1&pageSize=10"
```

### 2. 下发患者
```bash
curl -X POST -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"patientIds": [1, 2, 3]}' \
  "http://localhost:3000/api/inpatient/patients/assign"
```

### 3. 切换模板状态
```bash
curl -X PUT -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"isTemplate": true}' \
  "http://localhost:3000/api/inpatient/patients/1/toggle-template"
```

## 常见问题

### Q1: 下发后学生看不到患者？
**检查项**：
1. 确认患者确实是管理员创建的（doctorId = 管理员ID）
2. 确认patient_assignments表中有记录
3. 清除浏览器缓存后重新登录

### Q2: 管理员能看到其他用户的患者？
**原因**：可能是旧数据或缓存
**解决**：
1. 确认当前登录的是管理员账号
2. 检查数据库中患者的doctorId字段
3. 硬刷新页面（Ctrl+F5）

### Q3: 下发时报错？
**检查项**：
1. 确认使用的是管理员账号
2. 确认选中的患者存在且属于管理员
3. 查看后端控制台错误日志
4. 检查数据库中是否有非管理员用户

### Q4: 模板标记不显示？
**检查项**：
1. 确认当前不是管理员账号
2. 确认patient_assignments表中isTemplate字段为true
3. 检查前端是否正确接收了isTemplate字段

## 性能测试建议

### 大规模数据测试
1. 创建100个用户（50学生 + 50教师）
2. 管理员创建50个患者
3. 一次性下发所有患者
4. 观察：
   - 下发操作的响应时间
   - 学生/教师查询患者列表的响应时间
   - 数据库CPU和内存使用情况

### 并发测试
1. 多个用户同时查询患者列表
2. 观察系统响应时间和稳定性
