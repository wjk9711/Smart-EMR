# 患者下发数据隔离方案

## 需求分析

### 核心需求
当管理员下发患者及病案后，每个接收用户应该拥有：
1. **独立的患者数据副本** - 可以独立修改患者基本信息
2. **独立的病案数据副本** - 可以独立编辑病案内容
3. **互不影响的修改** - 用户A的修改不会影响用户B看到的数据
4. **可追溯性** - 能够追溯到原始数据来源

### 业务场景
```
管理员创建患者"小丽" (ID=5)
    ↓
管理员下发给 student1 和 teacher001
    ↓
student1 看到 "小丽" 的副本 (ID=100, sourcePatientId=5)
teacher001 看到 "小丽" 的副本 (ID=101, sourcePatientId=5)
    ↓
student1 修改患者电话 -> 只影响 ID=100
teacher001 修改患者电话 -> 只影响 ID=101
原始患者 ID=5 保持不变
```

## 方案设计

### 方案一：完全复制（推荐）⭐

#### 优点
- ✅ 完全隔离，互不影响
- ✅ 性能最好，查询简单
- ✅ 符合教学场景（每个学生独立完成病历）

#### 缺点
- ❌ 数据冗余
- ❌ 存储空间增加

#### 实现方式

**1. 修改 patient_assignments 表**

添加字段记录复制后的患者ID：
```sql
ALTER TABLE patient_assignments 
ADD COLUMN copiedPatientId INT NULL COMMENT '复制后的患者ID',
ADD COLUMN copiedAt DATETIME NULL COMMENT '复制时间';
```

**2. 下发时自动复制患者和病案**

```typescript
// 伪代码
async function assignPatientsToAllUsers(patientIds, userIds) {
  for (const patientId of patientIds) {
    // 1. 获取原始患者
    const originalPatient = await InpatientPatient.findByPk(patientId)
    
    for (const userId of userIds) {
      // 2. 复制患者数据
      const copiedPatient = await InpatientPatient.create({
        ...originalPatient.toJSON(),
        id: undefined, // 清除ID，让数据库自动生成
        doctorId: userId, // 设置为接收者的doctorId
        inpatientNo: generateUniqueInpatientNo(), // 生成新的唯一住院号
        sourcePatientId: patientId, // 记录来源
        createdAt: new Date(),
        updatedAt: new Date(),
      })
      
      // 3. 复制该患者的所有病案
      const originalRecords = await InpatientRecord.findAll({
        where: { patientId: patientId }
      })
      
      for (const record of originalRecords) {
        await InpatientRecord.create({
          ...record.toJSON(),
          id: undefined,
          patientId: copiedPatient.id, // 关联到复制的患者
          doctorId: userId, // 设置为接收者
          caseNo: copiedPatient.inpatientNo, // 使用新患者的住院号
          sourceRecordId: record.id, // 记录来源
          createdAt: new Date(),
          updatedAt: new Date(),
        })
      }
      
      // 4. 创建分配记录
      await PatientAssignment.create({
        patientId: patientId, // 原始患者ID
        userId: userId,
        assignedBy: adminId,
        copiedPatientId: copiedPatient.id, // 复制后的患者ID
        copiedAt: new Date(),
      })
    }
  }
}
```

### 方案二：视图隔离（不推荐）

#### 原理
- 不复制数据，通过权限控制实现逻辑隔离
- 每次查询时动态过滤

#### 缺点
- ❌ 修改仍然会相互影响
- ❌ 复杂度高，容易出错
- ❌ 不符合"独立修改"的需求

### 方案三：版本控制（过度设计）

#### 原理
- 保留单一患者记录
- 每次修改创建新版本
- 每个用户维护自己的版本链

#### 缺点
- ❌ 实现复杂
- ❌ 查询性能差
- ❌ 对于教学场景过于复杂

#### 后端实现要点

**1. 下发时复制数据**

```typescript
export const assignPatientsToAllUsers = async (req: AuthRequest, res: Response) => {
  const { patientIds, isTemplate = false } = req.body
  const currentUser = req.user!
  
  // 验证权限
  if (currentUser.roleType !== 'admin') {
    return error(res, 403, '无权操作')
  }
  
  const users = await User.findAll({
    where: { roleType: { [Op.ne]: 'admin' } },
    attributes: ['id'],
  })
  
  const sequelize = require('../config/database').default
  const transaction = await sequelize.transaction()
  
  try {
    for (const patientId of patientIds) {
      // 获取原始患者
      const originalPatient = await InpatientPatient.findByPk(patientId, { transaction })
      
      for (const user of users) {
        // 检查是否已经分配过
        const existing = await PatientAssignment.findOne({
          where: { patientId, userId: user.id },
          transaction,
        })
        
        if (existing) continue // 已存在则跳过
        
        // 生成新的唯一住院号
        const newInpatientNo = await generateUniqueInpatientNo()
        
        // 复制患者
        const copiedPatient = await InpatientPatient.create({
          name: originalPatient.name,
          gender: originalPatient.gender,
          birthDate: originalPatient.birthDate,
          age: originalPatient.age,
          idCard: originalPatient.idCard,
          phone: originalPatient.phone,
          address: originalPatient.address,
          inpatientNo: newInpatientNo, // 新的住院号
          department: originalPatient.department,
          bedNo: originalPatient.bedNo,
          admissionDate: originalPatient.admissionDate,
          dischargeDate: originalPatient.dischargeDate,
          status: originalPatient.status,
          diagnosis: originalPatient.diagnosis,
          doctorId: user.id, // 设置为接收者
          sourcePatientId: patientId, // 记录来源
        }, { transaction })
        
        // 复制病案
        const originalRecords = await InpatientRecord.findAll({
          where: { patientId },
          transaction,
        })
        
        for (const record of originalRecords) {
          await InpatientRecord.create({
            patientId: copiedPatient.id,
            recordType: record.recordType,
            caseNo: newInpatientNo, // 使用新住院号
            content: record.content,
            diagnosis: record.diagnosis,
            prescription: record.prescription,
            doctorId: user.id,
            sourceRecordId: record.id, // 记录来源
            status: record.status,
          }, { transaction })
        }
        
        // 创建分配记录
        await PatientAssignment.create({
          patientId,
          userId: user.id,
          assignedBy: currentUser.id,
          isTemplate,
          copiedPatientId: copiedPatient.id,
          copiedAt: new Date(),
        }, { transaction })
      }
    }
    
    await transaction.commit()
    
    res.json({
      code: 200,
      message: '下发成功',
    })
  } catch (err) {
    await transaction.rollback()
    throw err
  }
}
```

**2. 生成唯一住院号**

```typescript
async function generateUniqueInpatientNo(): Promise<string> {
  let inpatientNo: string
  let exists = true
  
  while (exists) {
    // 格式：ZY + 年月日 + 4位随机数
    const date = new Date()
    const dateStr = date.getFullYear().toString() + 
                   (date.getMonth() + 1).toString().padStart(2, '0') + 
                   date.getDate().toString().padStart(2, '0')
    const random = Math.floor(Math.random() * 10000).toString().padStart(4, '0')
    inpatientNo = `ZY${dateStr}${random}`
    
    const existing = await InpatientPatient.findOne({
      where: { inpatientNo },
    })
    
    exists = !!existing
  }
  
  return inpatientNo
}
```

## 优势总结

### ✅ 完全隔离
- 每个用户拥有独立的数据副本
- 修改互不影响
- 符合教学场景需求

### ✅ 可追溯
- 通过`sourcePatientId`和`sourceRecordId`可以追溯原始数据
- 便于审计和问题排查

### ✅ 性能良好
- 查询简单直接（WHERE doctorId = ?）
- 无需复杂的JOIN或子查询
- 索引效率高

### ✅ 数据安全
- 每个用户只能访问自己的数据
- 天然的数据隔离
- 符合最小权限原则
