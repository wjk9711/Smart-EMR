# 病案预览布局保持说明

## ✅ 核心要求

**保持原有病案首页布局不变的基础上，将所有类型的病案放在同一个界面垂直拼接显示。**

---

## 🎯 实现方案

### 1. 双模式支持

为预览组件添加了 `mode` 属性，支持两种显示模式：

```typescript
interface Props {
  visible: boolean
  recordData?: any
  mode?: 'dialog' | 'embedded'  // 显示模式
}
```

- **`mode="dialog"`（默认）**：对话框模式，用于单个病案预览
- **`mode="embedded"`**：内嵌模式，用于统一预览列表

### 2. 保持原始布局

**关键改进**：使用 Vue SFC 的 `template` 字符串保持原始的 HTML 表格布局结构。

### 3. 条件渲染模板

```vue
<template>
  <!-- 内嵌模式：直接显示内容，不带对话框 -->
  <div v-if="mode === 'embedded'" class="home-page-preview">
    <PreviewContent />
  </div>
  
  <!-- 对话框模式：使用 el-dialog -->
  <el-dialog
    v-else
    :model-value="visible"
    title="病案首页预览"
    width="1000px"
    top="2vh"
    @update:model-value="$emit('update:visible', $event)"
  >
    <div class="home-page-preview" ref="printArea">
      <PreviewContent />
    </div>
    
    <template #footer>
      <el-button @click="$emit('update:visible', false)">关闭</el-button>
      <el-button type="primary" @click="handlePrint">打印</el-button>
    </template>
  </el-dialog>
</template>
```

```vue
<!-- InpatientRecordList.vue -->
<div v-if="previewRecords.length > 0">

      </el-tag>

    </div>

    <!-- 病案首页预览 - 保持原有表格布局 -->
    <div v-if="record.recordType === 'home_page'" class="record-content">
      <HomePagePreview 
        :visible="true"
        :record-data="record"
        mode="embedded"  <!-- ✅ 使用内嵌模式 -->
      />
    </div>
    
    <!-- 入院记录预览 - 保持原有卡片布局 -->
    <div v-else-if="record.recordType === 'admission'" class="record-content">
      <AdmissionRecordPreview 
        :visible="true"
        :record-data="record"
        mode="embedded"  <!-- ✅ 使用内嵌模式 -->
      />
    </div>
    
    <!-- 其他类型病案使用 iframe -->
    <div v-else class="record-content other-record">
      <iframe 
        :src="`/inpatient/record/${record.id}?preview=true`"
        frameborder="0"
        style="width: 100%; min-height: 800px;"
      ></iframe>
    </div>

    <!-- 分隔线（最后一个病案不显示） -->
    <el-divider v-if="index < previewRecords.length - 1" />
  </div>
</div>
```

## ✅ 优势总结

### 1. 保持原有布局 ✅

- **病案首页**：完整的表格化布局，符合医疗文书规范
- **入院记录**：卡片式布局，清晰易读
- **所有样式**：完全保留原有的 CSS 样式

### 2. 代码复用 ✅

- 同一套 `PreviewContent` 组件用于两种模式
- 避免重复代码，易于维护
- 使用 Vue SFC 的 `template` 字符串，保持模板语法

### 3. 灵活切换 ✅

- 单个预览：`mode="dialog"`（默认）
- 统一预览：`mode="embedded"`
- 无需修改组件内部逻辑

### 4. 响应式数据 ✅

- 使用 `computed` 确保数据响应式
- 自动解析 JSON 格式的 content
- 兼容不同的数据结构

### 5. 打印友好 ✅

- 每个组件都有独立的打印功能
- 统一预览支持"打印全部"
- 自动分页，样式保留
