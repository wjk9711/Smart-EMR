# 首次病程记录提交500错误修复报告

## 📋 问题描述

用户反馈：**点击首次病程记录的“确定”按钮提交时，提示 “Request failed with status code 500，服务器错误”**

**错误详情**：
```
ValidationErrorItem {
  message: 'case_no must be unique',
  type: 'unique violation',
  path: 'case_no',
  value: '2024051001',  // ← 使用了患者的住院号作为病案号
  ...
}
```

---

## 🔍 问题分析

### 根本原因

**病案号（caseNo）唯一约束冲突 + 前端错误赋值** ❌

#### 问题1：数据库唯一约束

在 `InpatientRecord` 模型中，`caseNo` 字段定义了唯一约束：

```typescript
// emr-backend/src/models/InpatientRecord.ts (第34-39行)
caseNo: {
  type: DataTypes.STRING(50),
  allowNull: false,
  unique: true,  // ← 唯一约束
  comment: '病案号',
},
```

#### 问题2：前端错误地将住院号赋值给病案号

在 `ProgressRecordForm.vue` 的 `loadPatientInfo` 函数中：

```typescript
// ProgressRecordForm.vue (第342-344行) - 修改前 ❌
case 'caseNo':
  value = patient.inpatientNo || ''  // ← 错误！将住院号赋值给病案号
  break
```

**概念混淆**：
- **住院号（inpatientNo）**：患者住院时的编号，如 `'2024051001'`
- **病案号（caseNo）**：病案的独立编号，应该唯一且由系统生成

#### 问题场景

1. **用户点击“新增病案-首次病程记录”**
2. **加载患者信息**：
   ```javascript
   patient.inpatientNo = '2024051001'  // 患者的住院号
   formData.caseNo = patient.inpatientNo  // ← 错误赋值！
   ```
3. **填写表单并提交**：
   ```javascript
   const submitData = {
     patientId: 123,
     caseNo: '2024051001',  // ← 使用了已存在的住院号
     recordType: 'progress',
     content: JSON.stringify(recordContent),
     status: 'draft',
   }
   ```
4. **数据库冲突**：
   - 如果该住院号已经被用作其他病案的病案号
   - 触发唯一约束错误 → HTTP 500

---

## ✅ 解决方案

### 方案1：前端不再从患者信息中获取 caseNo

在 `ProgressRecordForm.vue` 的 `loadPatientInfo` 函数中，修改 `caseNo` 的处理逻辑：

**修改前** ❌：
```typescript
case 'caseNo':
  value = patient.inpatientNo || ''  // ← 错误！将住院号赋值给病案号
  break
```

**修改后** ✅：
```typescript
// caseNo 不应该从患者信息中获取，由后端自动生成
case 'caseNo':
  value = ''  // 新增模式保持为空，让后端生成
  break
```

---

### 方案2：后端自动生成病案号

在 `createInpatientRecord` 控制器中添加自动生成逻辑：

**修改前** ❌：
```typescript
export const createInpatientRecord = async (req: AuthRequest, res: Response) => {
  try {
    const currentUser = req.user!
    
    req.body.doctorId = currentUser.id
    
    // 序列化content字段为JSON字符串
    if (req.body.content && typeof req.body.content === 'object') {
      req.body.content = JSON.stringify(req.body.content)
    }
    
    const record = await InpatientRecord.create(req.body)  // ← caseNo为空会失败
    res.json({
      code: 200,
      data: record,
      message: '创建成功',
    })
  } catch (err: any) {
    console.error('Create record error:', err)
    return error(res, 500, err.message)
  }
}
```

**修改后** ✅：
```typescript
export const createInpatientRecord = async (req: AuthRequest, res: Response) => {
  try {
    const currentUser = req.user!
    
    // 确保doctorId被设置（数据隔离）
    req.body.doctorId = currentUser.id
    
    // ✅ 如果没有提供caseNo，自动生成
    if (!req.body.caseNo || req.body.caseNo.trim() === '') {
      // 生成格式：CASE + 时间戳(6位) + 随机数(3位)
      const timestamp = Date.now().toString().slice(-6)
      const random = Math.floor(Math.random() * 1000).toString().padStart(3, '0')
      req.body.caseNo = `CASE${timestamp}${random}`
      console.log('✅ 自动生成病案号:', req.body.caseNo)
    } else {
      console.log('ℹ️ 使用前端提供的病案号:', req.body.caseNo)
    }
    
    // 序列化content字段为JSON字符串
    if (req.body.content && typeof req.body.content === 'object') {
      req.body.content = JSON.stringify(req.body.content)
    }
    
    console.log('创建病案 - 请求体:', req.body)
    
    const record = await InpatientRecord.create(req.body)
    res.json({
      code: 200,
      data: record,
      message: '创建成功',
    })
  } catch (err: any) {
    console.error('Create record error:', err)
    console.error('Error details:', err.errors)  // ✅ 添加详细错误日志
    return error(res, 500, err.message)
  }
}
```

## 📊 病案号生成规则

### 格式说明

```
CASE + 时间戳(6位) + 随机数(3位)
```

### 示例

| 生成时间 | 时间戳 | 随机数 | 病案号 |
|---------|--------|--------|--------|
| 2026-06-05 10:30:45.123 | 451123 | 456 | CASE451123456 |
| 2026-06-05 10:30:46.789 | 467789 | 012 | CASE467789012 |
| 2026-06-05 10:30:47.456 | 474456 | 999 | CASE474456999 |

### 特点

✅ **唯一性**：时间戳 + 随机数保证唯一  
✅ **简洁性**：总长度固定为 12 位（CASE + 6 + 3）  
✅ **可读性**：以 "CASE" 开头，易于识别  
✅ **高效性**：无需查询数据库，直接生成

## ✅ 总结

### 问题根源

- ❌ **前端错误赋值**：将患者的住院号（inpatientNo）赋值给病案号（caseNo）

- ❌ **唯一约束冲突**：使用了已存在的住院号作为病案号，触发数据库唯一约束

### 解决方案

#### 前端修复
- ✅ `loadPatientInfo` 函数中，`caseNo` 不再从患者信息中获取
- ✅ 新增模式下 `caseNo` 保持为空字符串 `''`
- ✅ 让后端负责生成唯一的病案号

#### 后端修复
- ✅ 检测空的 `caseNo` 并自动生成
- ✅ 生成格式：`CASE + 时间戳(6位) + 随机数(3位)`
- ✅ 添加详细错误日志，便于调试

### 效果

- ✅ 首次病程记录可以正常提交
- ✅ 每个病案都有唯一的病案号
- ✅ 不会再出现 500 错误
- ✅ 代码更加健壮和可维护
- ✅ 前后端双重保护，确保数据一致性
