# 患者下发功能实现报告

## 功能概述

本次更新为系统新增了**患者下发功能**，并优化了数据隔离机制，确保所有用户（包括管理员）只能访问授权范围内的患者数据。

## 核心功能

### 1. 数据隔离优化

#### 原有问题
- 教师和管理员可以看到所有用户的患者和病案
- 缺乏细粒度的数据访问控制

#### 优化方案
**所有角色统一遵循最小权限原则：**
- **学生**：只能查看自己创建的患者 + 管理员下发的患者
- **教师**：只能查看自己创建的患者 + 管理员下发的患者
- **管理员**：只能查看自己创建的患者（不能看到其他用户的病人）

### 2. 患者下发功能

#### 功能描述
管理员可以将自己创建的患者批量下发给所有非管理员用户（学生和教师），实现教学场景中的患者资源共享。

#### 操作流程
1. 管理员登录系统
2. 进入"住院患者列表"页面
3. 勾选需要下发的患者（支持多选）
4. 点击"下发患者"按钮
5. 确认操作后，系统自动将选中患者分配给所有学生和教师

#### 前端界面
- **下发按钮**：仅在管理员视角显示，位于"新增患者"按钮左侧
- **选择框**：表格第一列为多选框，仅管理员可见
- **模板标记**：非管理员用户的患者列表中会显示"模板"标签，标识该患者是由管理员下发的

### 3. 数据库设计

#### 新增表：patient_assignments

```sql
CREATE TABLE `patient_assignments` (
  `id` INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  `patientId` INT NOT NULL COMMENT '患者ID',
  `userId` INT UNSIGNED NOT NULL COMMENT '用户ID（接收患者的用户）',
  `assignedBy` INT UNSIGNED NOT NULL COMMENT '分配者ID（管理员）',
  `isTemplate` TINYINT(1) DEFAULT false COMMENT '是否设为模板',
  `createdAt` DATETIME NOT NULL,
  `updatedAt` DATETIME NOT NULL,
  
  -- 外键约束
  FOREIGN KEY (`patientId`) REFERENCES `inpatient_patients` (`id`) ON DELETE CASCADE,
  FOREIGN KEY (`userId`) REFERENCES `users` (`id`) ON DELETE CASCADE,
  FOREIGN KEY (`assignedBy`) REFERENCES `users` (`id`) ON DELETE CASCADE,
  
  -- 索引
  UNIQUE INDEX `unique_patient_user_assignment` (`patientId`, `userId`),
  INDEX `idx_patient_assignments_user` (`userId`),
  INDEX `idx_patient_assignments_patient` (`patientId`)
) ENGINE=InnoDB;
```

#### 索引设计说明
- **联合唯一索引**：防止同一患者重复分配给同一用户
- **用户ID索引**：加速查询某用户的所有分配记录
- **患者ID索引**：加速查询某患者的所有分配记录

## 数据安全与一致性

### 1. 权限验证
- 所有下发操作均需验证管理员身份
- 只能下发自己创建的患者
- 病案查询基于患者ID进行过滤

### 2. 事务处理
- 批量分配使用数据库事务
- 失败时自动回滚，保证数据一致性

### 3. 重复保护
- 联合唯一索引防止重复分配
- `ignoreDuplicates: true` 跳过已存在的记录

### 4. 级联删除
- 删除患者时自动删除相关分配记录
- 删除用户时自动删除相关分配记录
