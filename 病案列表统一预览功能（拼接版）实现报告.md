<el-dialog
  v-model="previewAllVisible"

          </el-tag>
          <span class="record-index">{{ index + 1 }} / {{ previewRecords.length }}</span>
        </div>

        <!-- 病案首页预览 -->

            :record-data="record"
          />
        </div>
        
        <!-- 其他类型病案使用 iframe -->

      <el-empty description="暂无可预览的病案" />
    </div>

      <el-icon><Printer /></el-icon>

### 3. 打印功能

#### 打印全部病案函数

```typescript
// 打印全部病案
const handlePrintAll = () => {
  const printContent = previewAllContent.value?.innerHTML
  if (printContent) {
    const printWindow = window.open('', '_blank')
    if (printWindow) {
      printWindow.document.write(`
        <!DOCTYPE html>
        <html>
        <head>
          <meta charset="UTF-8">
          <title>病案预览打印</title>
          <style>
            body { 
              font-family: 'Microsoft YaHei', sans-serif; 
              padding: 20px;
              margin: 0;
            }
            .preview-record-section {
              margin-bottom: 40px;
              page-break-after: always;  // 每个病案后分页
            }
            .record-type-header {
              display: flex;
              align-items: center;
              gap: 15px;
              margin-bottom: 20px;
              padding-bottom: 10px;
              border-bottom: 2px solid #409eff;
            }
            .record-index {
              font-size: 14px;
              color: #909399;
            }
            @media print {
              .preview-record-section {
                page-break-after: always;
              }
            }
          </style>
        </head>
        <body>
          ${printContent}
        </body>
        </html>
      `)
      printWindow.document.close()
      setTimeout(() => {
        printWindow.print()
      }, 500)
    }
  } else {
    ElMessage.warning('没有可打印的内容')
  }
}
```

**关键特性**：
- ✅ **自动分页**：每个病案单独一页（`page-break-after: always`）
- ✅ **完整样式**：保留病案类型标题和序号
- ✅ **延迟打印**：等待 iframe 内容加载完成（500ms）

## 💡 技术要点

### 1. iframe 高度自适应

```vue
<iframe 
  :src="`/inpatient/record/${record.id}?preview=true`"
  frameborder="0"
  style="width: 100%; min-height: 800px;"
></iframe>
```

**问题**：iframe 内容高度不确定

**解决方案**：
- 设置 `min-height: 800px` 保证最小高度
- 实际高度由 iframe 内部内容决定
- 可通过 postMessage 实现更精确的高度控制（可选优化）

### 2. 打印分页控制

```css
.preview-record-section {
  page-break-after: always;
}

@media print {
  .preview-record-section {
    page-break-after: always;
  }
}
```

**作用**：
- 每个病案单独打印在一页
- 避免病案内容被分割到两页
- 提升打印效果的专业性

### 3. 延迟打印

```typescript
setTimeout(() => {
  printWindow.print()
}, 500)
```

**原因**：
- iframe 内容需要时间加载
- 立即打印可能导致内容为空
- 500ms 延迟平衡了速度和完整性

## ⚠️ 注意事项

### 1. iframe 加载性能

**问题**：多个 iframe 同时加载可能影响性能

**优化建议**：
```typescript
// 懒加载：只加载可见区域的 iframe
const visibleRecords = ref<Set<number>>(new Set())

const handleScroll = () => {
  // 检测哪些病案在可视区域内
  // 只加载这些病案的 iframe
}
```

### 2. 内存管理

**问题**：大量病案可能导致内存占用过高

**解决方案**：
```typescript
// 关闭预览时清理 iframe
const handleClosePreview = () => {
  previewRecords.value = []
  currentPreviewIndex.value = 0
  currentPreviewUrl.value = ''
  // 强制销毁 DOM
  nextTick(() => {
    previewAllContent.value = null
  })
}
```

### 3. 打印样式兼容性

**问题**：不同浏览器打印样式可能有差异

**建议**：
- 测试 Chrome、Firefox、Edge 等主流浏览器
- 使用 `@media print` 确保打印样式独立
- 提供"导出 PDF"作为备选方案

### 4. 移动端适配

**问题**：90% 宽度在移动端可能过宽

**优化**：
```scss
@media (max-width: 768px) {
  .preview-all-container {
    padding: 10px;
    
    .record-type-header {
      flex-direction: column;
      align-items: flex-start;
    }
  }
}
```
