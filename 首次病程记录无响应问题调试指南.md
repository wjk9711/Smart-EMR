# 首次病程记录无响应问题调试指南

## 📋 问题描述

用户反馈：**点击"新增首次病程记录"无任何响应**

---

## 🔍 可能的原因

### 1. patientId 为空 ❌

最常见的原因是从住院患者列表进入病案列表时，没有正确传递 `patientId` 参数。

**检查方法**：
- 打开浏览器控制台（F12）
- 查看 Console 标签页
- 点击"新增病案-首次病程记录"
- 查看输出的日志

**预期输出**：
```
点击新增病案 - 类型: progress
当前 patientId: undefined  ← 问题在这里！
当前 route.query: { patientId: undefined }
patientId 为空，无法创建病案
```

**解决方案**：
确保从住院患者列表跳转到病案列表时，URL 包含 `patientId` 参数：
```
/inpatient/records?patientId=123&patientName=张三
```

---

### 2. 路由参数解析错误 ❌

如果 URL 中有 `patientId`，但解析失败。

**检查方法**：
在控制台输入：
```javascript
console.log(window.location.search)
console.log(new URLSearchParams(window.location.search).get('patientId'))
```

**预期输出**：
```
"?patientId=123&patientName=张三"
"123"
```

如果输出是 `null` 或 `undefined`，说明路由参数没有正确传递。

---

### 3. 组件未正确挂载 ❌

ProgressRecordForm 组件可能没有正确导入或注册。

**检查方法**：
查看控制台是否有以下错误：
```
Failed to resolve component: ProgressRecordForm
```

**解决方案**：
检查 InpatientRecordList.vue 中是否正确导入：
```typescript
import ProgressRecordForm from './ProgressRecordForm.vue'
```

---

### 4. v-model 绑定问题 ❌

`showProgressForm` 的状态变化可能没有正确触发组件更新。

**检查方法**：
在控制台查看日志：
```
准备打开表单 - showProgressForm: false
已设置 showProgressForm = true
ProgressRecordForm - visible 变化: true
```

如果看到 "已设置 showProgressForm = true" 但没有看到 "ProgressRecordForm - visible 变化: true"，说明 v-model 绑定有问题。

---

## 🛠️ 常见修复方案

### 修复1: 确保正确跳转

从住院患者列表跳转到病案列表时：

```typescript
// 正确的跳转方式
router.push({
  path: '/inpatient/records',
  query: {
    patientId: patient.id,
    patientName: patient.name
  }
})
```

**不要**直接导航到 `/inpatient/records` 而不带参数。

---

### 修复2: 添加容错处理

如果 patientId 为空，提供更友好的提示：

```typescript
const handleCreateByType = (type: string) => {
  if (!patientId.value) {
    ElMessageBox.confirm(
      '当前未选择患者，是否返回患者列表选择？',
      '提示',
      {
        confirmButtonText: '返回患者列表',
        cancelButtonText: '取消',
        type: 'warning'
      }
    ).then(() => {
      router.push('/inpatient/patients')
    })
    return
  }
  
  // ... 原有逻辑
}
```

---

### 修复3: 检查路由配置

确保路由配置正确：

```typescript
// router/index.ts
{
  path: '/inpatient/records',
  name: 'InpatientRecordList',
  component: () => import('@/views/inpatient/InpatientRecordList.vue'),
  meta: { requiresAuth: true }
}
```
