# 教师查看学生病案权限修复报告

## 🐛 问题描述

**用户反馈**：教师用户在住院患者列表中看不到学生创建的病人"小丽"，因此无法查看该患者的病案列表。

**根本原因**：数据隔离策略过于严格，限制了教师的查看权限。

---

## 🔍 问题分析

### 问题1: 住院患者列表权限限制

**文件**: `emr-backend/src/controllers/inpatientController.ts`  
**位置**: 第7-45行 `getInpatientPatients` 函数

**原代码逻辑**:
```typescript
// ❌ 错误：所有非管理员用户（包括教师）只能看到自己创建的患者
if (currentUser.roleType !== 'admin') {
  whereCondition.doctorId = currentUser.id
}
```

**问题**:
- 学生创建了患者"小丽"（doctorId = 学生ID）
- 教师登录後，查询条件 `doctorId = 教师ID`
- 结果：教师看不到"小丽"
- 影响：教师无法进入患者的病案列表

---

### 问题2: 病案列表权限限制

**文件**: `emr-backend/src/controllers/inpatientController.ts`  
**位置**: 第132-197行 `getInpatientRecords` 函数

**原代码逻辑**:
```typescript
// ❌ 错误：教师只能查看已提交的病案
if (currentUser.roleType === 'teacher') {
  whereCondition.submitStatus = { 
    [Symbol.for('in')]: ['pending_review', 'passed', 'rejected'] 
  }
}
```

**问题**:
- 学生新建病案时，默认状态是 `not_submitted`（未提交）
- 教师查询条件排除了 `not_submitted` 状态的病案
- 结果：教师看不到未提交的病案
- 影响：即使进入病案列表，也看不到新增的病案

---

## ✅ 修复方案

### 修复1: 放宽住院患者列表权限

**修改位置**: 第12-18行

**新代码**:
```typescript
// ✅ 正确：学生只能看自己的患者，教师和管理员可以看所有患者
const whereCondition: any = {}

// 数据隔离：
// - 学生：只能查看自己创建的患者
// - 教师：可以查看所有患者（用于教学和质控）
// - 管理员：可以查看所有患者
if (currentUser.roleType === 'student') {
  whereCondition.doctorId = currentUser.id
}
// 教师和管理员不限制 doctorId，可以查看所有患者
```

**效果**:
- ✅ 学生：只能看到自己创建的患者（保持数据隔离）
- ✅ 教师：可以看到所有患者（便于教学监督）
- ✅ 管理员：可以看到所有患者（全局管理）

---

### 修复2: 放宽病案列表权限

**修改位置**: 第142-150行

**新代码**:
```typescript
// ✅ 正确：学生只能看自己的病案，教师和管理员可以看所有病案
const whereCondition: any = {}
if (patientId) whereCondition.patientId = patientId
if (caseNo) whereCondition.caseNo = { like: `%${caseNo}%` }
if (status) whereCondition.status = status

// 数据隔离：
// - 学生：只能查看自己创建的病案（所有状态）
// - 教师：可以查看所有病案（用于教学和质控）
// - 管理员：可以查看所有病案
if (currentUser.roleType === 'student') {
  whereCondition.doctorId = currentUser.id
}
// 教师和管理员不限制 doctorId，可以查看所有病案
```

**效果**:
- ✅ 学生：只能看到自己创建的病案（保持数据隔离）
- ✅ 教师：可以看到所有病案（包括未提交的）
- ✅ 管理员：可以看到所有病案（全局管理）

## 🎯 设计理念

### 数据隔离原则

**学生视角**:
- 只能管理自己的患者和病案
- 防止误操作他人的数据
- 保护隐私和数据安全

**教师视角**:
- 可以查看所有学生的患者和病案
- 便于教学监督和质控审核
- 全面了解学生学习情况

**管理员视角**:
- 全局管理权限
- 系统维护和数据分析

---

### 为什么这样设计？

1. **教学需求**
   - 教师需要了解学生的学习进度
   - 需要查看所有学生的作业（病案）
   - 包括未完成的作品（草稿）

2. **质控需求**
   - 教师需要在学生提交前进行指导
   - 不能等到提交后才发现问题
   - 提前介入可以提高教学质量

3. **管理需求**
   - 教师需要全面掌握班级情况
   - 了解哪些学生完成了任务
   - 哪些学生还需要帮助

---

## ⚠️ 注意事项

### 1. 学生之间的隔离仍然保持
- ✅ 学生A看不到学生B的患者
- ✅ 学生A看不到学生B的病案
- ✅ 保证公平竞争和隐私保护

### 2. 教师的查看权限不等于编辑权限
- ✅ 教师可以查看所有病案
- ⚠️ 但编辑权限需要通过其他方式控制（如：只能编辑自己创建的）
- 💡 建议：前端界面可以区分"查看"和"编辑"按钮

### 3. 提交状态的作用
- `not_submitted`: 草稿，学生正在编辑
- `pending_review`: 已提交，等待教师审核
- `passed`: 审核通过
- `rejected`: 审核不通过，需要修改

教师可以看到所有状态的病案，但质控操作主要针对 `pending_review` 状态的病案。
