### 问题3：修复患者基本信息未带入问题

#### 3.1 入院记录预览

**文件**: `AdmissionRecordPreview.vue`

**修改位置**: 第365-379行

**问题分析**：
- 原逻辑：`return content.admissionRecord || content || {}`
- 问题：患者基本信息（patientName、gender 等）存储在 `recordData` 顶层，不在 `admissionRecord` 对象中
- 结果：预览时患者信息显示为 "-"

**修改前** ❌：
```typescript
const data = computed(() => {
  if (!props.recordData) return {}
  
  let content = props.recordData.content
  if (typeof content === 'string') {
    try {
      content = JSON.parse(content)
    } catch (e) {
      console.error('Failed to parse content:', e)
      return {}
    }
  }
  
  return content.admissionRecord || content || {}
})
```

**修改后** ✅：
```typescript
const data = computed(() => {
  if (!props.recordData) return {}
  
  let content = props.recordData.content
  if (typeof content === 'string') {
    try {
      content = JSON.parse(content)
    } catch (e) {
      console.error('Failed to parse content:', e)
      return {}
    }
  }
  
  const admissionRecord = content.admissionRecord || content || {}
  
  // 合并患者基本信息（从 recordData 顶层获取）
  return {
    ...admissionRecord,
    // 患者基本信息优先从 recordData 顶层获取
    patientName: props.recordData.patientName || admissionRecord.patientName || '-',
    gender: props.recordData.gender || admissionRecord.gender || '-',
    age: props.recordData.age || admissionRecord.age || '',
    inpatientNo: props.recordData.inpatientNo || admissionRecord.inpatientNo || '-',
    department: props.recordData.department || admissionRecord.department || '-',
    bedNo: props.recordData.bedNo || admissionRecord.bedNo || '-',
    admissionDate: props.recordData.admissionDate || admissionRecord.admissionDate || '',
    caseNo: props.recordData.caseNo || admissionRecord.caseNo || '-',
  }
})
```

## 🎯 技术要点

### 1. 数据合并策略

使用展开运算符 `...` 合并数据：

```typescript
return {
  ...admissionRecord,  // 先展开病案特有字段
  // 然后覆盖患者基本信息（优先从 recordData 顶层获取）
  patientName: props.recordData.patientName || admissionRecord.patientName || '-',
  // ...
}
```

**优先级**：
1. `props.recordData.patientName` - 从 recordData 顶层获取（最高优先级）
2. `admissionRecord.patientName` - 从病案内容中获取（备用）
3. `'-'` - 默认值（兜底）

---

### 2. 需要合并的字段

| 字段 | 说明 | 来源 |
|------|------|------|
| patientName | 患者姓名 | recordData 顶层 |
| gender | 性别 | recordData 顶层 |
| age | 年龄 | recordData 顶层 |
| inpatientNo | 住院号 | recordData 顶层 |
| department | 科室 | recordData 顶层 |
| bedNo | 床号 | recordData 顶层 |
| admissionDate | 入院日期 | recordData 顶层 |
| caseNo | 病案号 | recordData 顶层 |

---

### 3. 为什么需要这样处理？

**数据结构分析**：

```javascript
// recordData 的结构
{
  id: 123,
  patientId: 456,
  caseNo: "CASE123456",
  recordType: "admission",
  content: '{"admissionRecord": {...}}',  // JSON 字符串
  // 以下字段在 recordData 顶层
  patientName: "张三",
  gender: "男",
  age: 45,
  inpatientNo: "2024051001",
  department: "内科",
  bedNo: "101",
  admissionDate: "2024-05-10",
  ...
}
```

**问题**：
- `content.admissionRecord` 只包含病案特有字段（主诉、现病史等）
- 患者基本信息在 `recordData` 顶层，不在 `content.admissionRecord` 中
- 如果只返回 `content.admissionRecord`，患者信息会丢失

**解决**：
- 合并两个数据源
- 优先使用 `recordData` 顶层的患者信息
- 备用使用 `admissionRecord` 中的患者信息（如果有）

## 💡 经验教训

### 1. 数据结构理解

- **必须清楚数据的存储位置**
- `recordData` 顶层 vs `content.xxx` 内部
- 不同层级的数据需要正确合并

### 2. 防御性编程

- 使用多级 fallback：`A || B || '-'`
- 确保即使某层数据缺失，也不会报错
- 提供合理的默认值

### 3. 代码注释

- 添加清晰的注释说明数据来源
- 解释为什么要这样合并
- 方便后续维护和理解
