#### 2.1 添加病案类型名称映射函数

```typescript
// 病案类型名称映射
const getRecordTypeName = (recordType: string): string => {
  const typeMap: Record<string, string> = {
    home_page: '病案首页',
    admission: '入院记录',
    progress: '首次病程记录',
    operation: '手术记录',
    discharge: '出院记录',
  }
  return typeMap[recordType] || recordType
}
```

#### 2.2 添加重复检查逻辑

```typescript
// 检查该患者是否已有该类型的病案
const existingRecord = await InpatientRecord.findOne({
  where: {
    patientId: req.body.patientId,
    recordType: req.body.recordType,
  },
})

if (existingRecord) {
  console.log('❌ 病案类型重复:', req.body.recordType)
  return error(res, 400, `该患者已存在${getRecordTypeName(req.body.recordType)}，不能重复创建`)

```

**业务规则**:
- 每个患者在同一个病案类型下只能有一个病案
- 例如：一个患者只能有一个"病案首页"，不能创建第二个
- 如果尝试创建重复类型的病案，返回 400 错误并提示用户

**错误提示示例**:
- "该患者已存在病案首页，不能重复创建"
- "该患者已存在入院记录，不能重复创建"
- "该患者已存在首次病程记录，不能重复创建"

### 需求3：病案列表按类型排序

**状态**: ✅ 已完成

**实现位置**: [inpatientController.ts](file://f:/File/Project/EMR/emr-backend/src/controllers/inpatientController.ts#L28-L40) 和 [inpatientController.ts](file://f:/File/Project/EMR/emr-backend/src/controllers/inpatientController.ts#L203-L209)

#### 3.1 添加排序权重函数

```typescript
// 病案类型排序权重（用于自定义排序）
const getRecordTypeSortWeight = (recordType: string): number => {
  const weightMap: Record<string, number> = {
    home_page: 1,      // 病案首页
    admission: 2,      // 入院记录
    progress: 3,       // 首次病程记录
    operation: 4,      // 手术记录
    discharge: 5,      // 出院记录
  }
  return weightMap[recordType] || 99
}
```

#### 3.2 查询后排序

```typescript
// 按病案类型排序：病案首页 -> 入院记录 -> 首次病程记录 -> 手术记录 -> 出院记录
recordsWithPatient.sort((a: any, b: any) => {
  const weightA = getRecordTypeSortWeight(a.recordType)
  const weightB = getRecordTypeSortWeight(b.recordType)
  return weightA - weightB
})
```

**排序规则**:
1. **病案首页** (权重 1) - 最先显示
2. **入院记录** (权重 2)
3. **首次病程记录** (权重 3)
4. **手术记录** (权重 4)
5. **出院记录** (权重 5) - 最后显示

**说明**:
- 由于 Sequelize 的 `order` 不支持自定义排序函数，因此在查询后对结果进行排序
- 相同类型的病案保持原有的创建时间顺序
- 未知类型的病案权重为 99，排在最后
