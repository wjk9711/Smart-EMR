# 404错误快速修复指南

## 问题描述

访问 `http://47.97.200.237` 时出现：
```
POST http://47.97.200.237/api/auth/login 404 (Not Found)
```

## 根本原因

**Vite的代理配置只在开发环境有效，生产环境构建后的前端代码直接请求服务器API路径，但后端服务可能：**
1. 没有运行
2. 没有监听正确的网络接口（只监听localhost）
3. 端口被防火墙阻止
4. 前端API地址配置不正确

## 已实施的修复

### 1. 前端环境变量支持 ✅

创建了环境变量配置文件：
- `.env.development` - 开发环境使用 `http://localhost:3000`
- `.env.production` - 生产环境使用 `http://47.97.200.237:3000`

修改了 `src/utils/request.ts`：
```typescript
baseURL: import.meta.env.VITE_API_BASE_URL ? `${import.meta.env.VITE_API_BASE_URL}/api` : '/api'
```

### 2. 后端监听所有网络接口 ✅

修改了 `emr-backend/src/app.ts`：
```typescript
const port = typeof config.port === 'string' ? parseInt(config.port) : config.port;
app.listen(port, '0.0.0.0', () => {
  console.log(`Server is running on port ${port}`);
});
```

## 立即执行的修复步骤


```bash

# 确认 .env.production 中的地址正确
```

### 步骤3：部署前端文件

将 `emr-frontend/dist` 目录上传到服务器的Web目录：

```bash
# 方法1: 使用scp
scp -r emr-frontend/dist/* root@47.97.200.237:/var/www/html/

# 方法2: 使用FTP工具（如FileZilla）
# 连接服务器后上传dist目录内容
```

### 步骤4：配置防火墙

```bash
# Ubuntu/Debian
sudo ufw allow 3000/tcp
sudo ufw reload

# CentOS/RHEL
sudo firewall-cmd --add-port=3000/tcp --permanent
sudo firewall-cmd --reload

# 阿里云：在控制台配置安全组
# 添加入站规则：TCP 3000端口，授权对象 0.0.0.0/0
```

### 步骤5：验证部署

```bash

# 测试登录接口
```

## 常见问题排查

### 问题1：后端服务未启动

**症状：** `curl http://localhost:3000/health` 失败

**解决：**
```bash
cd /path/to/emr-backend
npm run build
nohup node dist/app.js > backend.log 2>&1 &
tail -f backend.log  # 查看日志
```

### 问题2：端口被占用

**症状：** 启动时报错 `EADDRINUSE`

**解决：**
```bash
# 查找占用3000端口的进程
lsof -i :3000
# 或
netstat -tlnp | grep 3000

# 杀死进程
kill -9 <PID>

# 或使用其他端口
PORT=3001 node dist/app.js
```

### 问题3：防火墙阻止

**症状：** 本地可以访问 `http://localhost:3000`，但外网无法访问

**解决：**
```bash
# 检查防火墙状态
sudo ufw status
# 或
sudo firewall-cmd --list-all

# 开放端口
sudo ufw allow 3000/tcp
# 或
sudo firewall-cmd --add-port=3000/tcp --permanent && sudo firewall-cmd --reload
```

### 问题4：阿里云安全组

**症状：** 防火墙已开放，但仍无法访问

**解决：**
1. 登录阿里云控制台
2. 进入 ECS实例 -> 安全组
3. 添加入站规则：
   - 端口范围：3000/3000
   - 协议类型：TCP
   - 授权对象：0.0.0.0/0
   - 优先级：1

### 问题5：CORS错误

**症状：** 浏览器控制台显示CORS错误

**解决：** 修改 `emr-backend/src/app.ts`：
```typescript
app.use(cors({
  origin: ['http://47.97.200.237', 'http://your-domain.com'],
  credentials: true
}))
```

## 推荐的长期方案：Nginx反向代理

为了更稳定的生产环境，建议使用Nginx：

### 安装Nginx
```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nginx

# CentOS/RHEL
sudo yum install nginx
```

### 配置Nginx

创建配置文件 `/etc/nginx/sites-available/emr`：

```nginx
server {
    listen 80;
    server_name 47.97.200.237;

    # 前端静态文件
    location / {
        root /var/www/html;
        try_files $uri $uri/ /index.html;
        index index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://localhost:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 启用配置
```bash
sudo ln -s /etc/nginx/sites-available/emr /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 修改前端配置

如果使用Nginx，前端可以直接使用相对路径：

`.env.production`:
```env
VITE_API_BASE_URL=
```

这样前端会请求同域下的 `/api`，由Nginx代理到后端。
