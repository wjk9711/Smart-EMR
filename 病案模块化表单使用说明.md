# 病案类型模块化表单使用说明

## 📋 概述

根据**方案B：增量开发策略**，我们为每种病案类型创建了独立的表单组件，确保不同类型病案的差异化编辑界面。

---

## 📁 文件清单

### 新增的独立表单组件

1. **AdmissionRecordForm.vue** - 入院记录表单
   - 路径: `emr-frontend/src/views/inpatient/AdmissionRecordForm.vue`
   - 行数: 578行

2. **ProgressRecordForm.vue** - 首次病程记录表单
   - 路径: `emr-frontend/src/views/inpatient/ProgressRecordForm.vue`
   - 行数: 449行

3. **DischargeRecordForm.vue** - 出院记录表单
   - 路径: `emr-frontend/src/views/inpatient/DischargeRecordForm.vue`
   - 行数: 432行

4. **OperationRecordForm.vue** - 手术记录表单
   - 路径: `emr-frontend/src/views/inpatient/OperationRecordForm.vue`

## 🎯 各表单模块特点

### 1. 入院记录 (AdmissionRecordForm.vue)

**特有字段**:
- ✅ 主诉 (chiefComplaint)
- ✅ 现病史 (presentIllness)
- ✅ 既往史 (pastHistory)
- ✅ 个人史 (personalHistory)
- ✅ 婚育史 (marriageHistory)
- ✅ 家族史 (familyHistory)
- ✅ 生命体征（体温、脉搏、呼吸、血压）
- ✅ 一般情况 (generalExam)
- ✅ 专科检查 (specialistExam)
- ✅ 实验室检查 (labExam)
- ✅ 影像学检查 (imagingExam)
- ✅ 入院诊断 + ICD-10编码
- ✅ 诊疗计划

**适用场景**: 患者入院时填写的完整病历记录

---

### 2. 首次病程记录 (ProgressRecordForm.vue)

**特有字段**:
- ✅ 记录时间 (recordDateTime) - 精确到分钟
- ✅ 病例特点 (caseFeatures)
- ✅ 初步诊断 + ICD-10编码
- ✅ 诊断依据 (diagnosisBasis)
- ✅ 鉴别诊断 (differentialDiagnosis)
- ✅ 诊疗计划 (treatmentPlan)
- ✅ 注意事项 (precautions)

**适用场景**: 入院后8小时内完成的首次病程记录

---

### 3. 出院记录 (DischargeRecordForm.vue)

**特有字段**:
- ✅ 出院日期 (dischargeDate)
- ✅ 入院情况 (admissionCondition)
- ✅ 入院诊断 + ICD-10编码
- ✅ 诊疗经过 (treatmentCourse)
- ✅ 检查结果 (examinationResults)
- ✅ 出院诊断 + ICD-10编码
- ✅ 出院情况 (dischargeCondition)
- ✅ 离院方式 (dischargeMethod)
- ✅ 出院医嘱 (dischargeAdvice)
- ✅ 带药情况 (medications)
- ✅ 复诊计划 (followUpPlan)

**适用场景**: 患者出院时填写的总结性记录

---

### 4. 手术记录 (OperationRecordForm.vue)

**特有字段**:
- ✅ 手术日期 (operationDate)
- ✅ 手术名称 (operationName)
- ✅ 手术者 (operator)
- ✅ 助手 (assistants)
- ✅ 麻醉方式 (anesthesiaMethod)
- ✅ 麻醉医师 (anesthesiologist)
- ✅ 手术开始/结束时间
- ✅ 术前诊断 + ICD-10编码
- ✅ 术后诊断 + ICD-10编码
- ✅ 手术指征 (operationIndication)
- ✅ 手术经过 (operationProcedure)
- ✅ 术中发现 (intraoperativeFindings)
- ✅ 出血量 (bloodLoss)
- ✅ 输血情况 (bloodTransfusion)
- ✅ 标本送检 (specimenExam)
- ✅ 术后处理 (postoperativeCare)

**适用场景**: 手术后24小时内完成的手术记录

## 🔧 使用方法

### 在病案列表中调用不同表单

```vue
<template>
  <!-- 入院记录 -->
  <AdmissionRecordForm
    v-model:visible="showAdmissionForm"
    :record-data="currentRecord"
    :patient-id="patientId"
    @success="handleSuccess"
  />

  <!-- 首次病程记录 -->
  <ProgressRecordForm
    v-model:visible="showProgressForm"
    :record-data="currentRecord"
    :patient-id="patientId"
    @success="handleSuccess"
  />

  <!-- 出院记录 -->
  <DischargeRecordForm
    v-model:visible="showDischargeForm"
    :record-data="currentRecord"
    :patient-id="patientId"
    @success="handleSuccess"
  />

  <!-- 手术记录 -->
  <OperationRecordForm
    v-model:visible="showOperationForm"
    :record-data="currentRecord"
    :patient-id="patientId"
    @success="handleSuccess"
  />
</template>

<script setup lang="ts">
import { ref } from 'vue'
import AdmissionRecordForm from './AdmissionRecordForm.vue'
import ProgressRecordForm from './ProgressRecordForm.vue'
import DischargeRecordForm from './DischargeRecordForm.vue'
import OperationRecordForm from './OperationRecordForm.vue'

const showAdmissionForm = ref(false)
const showProgressForm = ref(false)
const showDischargeForm = ref(false)
const showOperationForm = ref(false)
const currentRecord = ref(null)
const patientId = ref(123)

// 打开入院记录表单
const openAdmissionForm = () => {
  currentRecord.value = null
  showAdmissionForm.value = true
}

// 打开首次病程记录表单
const openProgressForm = () => {
  currentRecord.value = null
  showProgressForm.value = true
}

// ... 其他表单类似
</script>
```

## ✨ 核心特性

### 1. 自动填充患者信息
- ✅ 从住院患者API自动获取基本信息
- ✅ 姓名、性别、年龄、住院号等字段自动填充
- ✅ 关键字段设为禁用状态，防止误修改

### 2. 病案号与住院号一致
- ✅ 新建时自动将病案号设置为住院号
- ✅ 确保数据一致性

### 3. ICD-10编码检查
- ✅ 入院记录、首次病程记录支持编码检查
- ✅ 调用通义千问API验证编码正确性
- ✅ 自动推荐正确的ICD-10编码

### 4. 表单验证
- ✅ 必填字段验证
- ✅ 实时错误提示
- ✅ 提交前完整校验

### 5. 数据持久化
- ✅ 所有内容存储在 `content` JSON字段中
- ✅ 符合方案B的数据结构规范
- ✅ 支持草稿保存和后续编辑

## 🔄 与原有系统的集成

### 保持向后兼容

1. **原有 InpatientRecordForm.vue 不变**
   - 继续使用通用表单
   - 不影响现有业务流程

2. **新增表单独立运行**
   - 不修改原有代码
   - 通过条件判断选择使用哪个表单

3. **数据库结构不变**
   - 继续使用 `content` TEXT字段存储JSON
   - 不同的 `recordType` 对应不同的JSON结构

### 集成建议

在 `InpatientRecordList.vue` 中添加病案类型选择：

```vue
<el-dropdown @command="handleCreateRecord">
  <el-button type="primary">
    新建病案<el-icon><ArrowDown /></el-icon>
  </el-button>
  <template #dropdown>
    <el-dropdown-menu>
      <el-dropdown-item command="admission">入院记录</el-dropdown-item>
      <el-dropdown-item command="progress">首次病程记录</el-dropdown-item>
      <el-dropdown-item command="discharge">出院记录</el-dropdown-item>
      <el-dropdown-item command="operation">手术记录</el-dropdown-item>
    </el-dropdown-menu>
  </template>
</el-dropdown>

<script setup>
const handleCreateRecord = (type: string) => {
  switch (type) {
    case 'admission':
      showAdmissionForm.value = true
      break
    case 'progress':
      showProgressForm.value = true
      break
    case 'discharge':
      showDischargeForm.value = true
      break
    case 'operation':
      showOperationForm.value = true
      break
  }
}
</script>
```
