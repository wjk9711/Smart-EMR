# 病案首页病案号显示问题修复报告

## 📅 修复日期
2026-06-08

## 🐛 问题描述

**用户反馈**：在病案列表中，病案类型为病案首页的病案号没有显示，应为患者的住院号

**问题现象**：
1. 新增病案首页并保存
2. 返回病案列表页面
3. 查看病案列表表格
4. **病案首页类型的"病案号"列显示为空**（或显示 `-`）
5. 但"住院号"列有正确显示

---

## 🔍 问题分析

### 根本原因

**数据显示逻辑问题**：病案列表直接显示 `caseNo` 字段，但对于病案首页类型，如果数据库中 `caseNo` 为空，应该显示 `inpatientNo`（住院号）。

#### 原有代码（有问题）

```vue
<!-- ❌ 直接显示 caseNo -->
<el-table-column prop="caseNo" label="病案号" width="120" />
```

#### 问题场景分析

**场景1：其他类型病案**
```
入院记录、病程记录等
  ↓
caseNo 字段有值（手动输入或自动生成）
  ↓
正常显示 ✅
```

**场景2：病案首页（问题场景）**
```
病案首页
  ↓
根据需求：病案号 = 住院号
  ↓
但可能由于以下原因导致 caseNo 为空：
  1. 创建时 formData.caseNo 未正确赋值
  2. 数据库保存失败
  3. 历史数据迁移问题
  ↓
列表显示为空 ❌
```

---

## ✅ 修复方案

### 修复策略

**智能显示逻辑**：
- 对于病案首页类型，如果 `caseNo` 为空，则显示 `inpatientNo`
- 对于其他类型，直接显示 `caseNo`

### 修复代码

#### 1. 修改表格列定义

**文件**: `emr-frontend/src/views/inpatient/InpatientRecordList.vue`

```vue
<!-- ✅ 修复后：使用自定义模板 -->
<el-table-column label="病案号" width="120">
  <template #default="{ row }">
    {{ getCaseNo(row) }}
  </template>
</el-table-column>
```

#### 2. 添加智能获取函数

```typescript
// ✅ 新增函数：智能获取病案号
const getCaseNo = (row: any) => {
  // 如果是病案首页类型，且 caseNo 为空，则显示住院号
  if (row.recordType === 'home_page' && (!row.caseNo || row.caseNo === '')) {
    return row.inpatientNo || '-'
  }
  return row.caseNo || '-'
}
```

## ⚠️ 注意事项

### 1. 数据一致性

虽然前端做了兼容处理，但**最佳实践是确保数据库中 `caseNo` 字段有值**。

**建议**：
- 在创建病案首页时，确保 `caseNo` 正确设置为住院号
- 检查后端创建接口是否正确保存了 `caseNo`
- 对于历史数据，可以执行数据迁移脚本补充 `caseNo`

### 2. 后端数据检查

可以通过以下 SQL 检查是否有病案首页的 `caseNo` 为空：

```sql
SELECT id, record_type, case_no, patient_id
FROM inpatient_records
WHERE record_type = 'home_page' 
  AND (case_no IS NULL OR case_no = '');
```

### 3. 数据迁移（可选）

如果发现大量历史数据的 `caseNo` 为空，可以执行迁移：

```sql
UPDATE inpatient_records r
JOIN inpatient_patients p ON r.patient_id = p.id
SET r.case_no = p.inpatient_no
WHERE r.record_type = 'home_page'
  AND (r.case_no IS NULL OR r.case_no = '');
```

### 4. 前端逻辑优先级

当前的显示逻辑优先级：
1. **病案首页 + caseNo 为空** → 显示 `inpatientNo`
2. **其他情况** → 显示 `caseNo`
3. **都为空** → 显示 `-`

如果需要调整优先级，可以修改 `getCaseNo` 函数。

## ✅ 总结

### 问题根源
- 病案列表直接显示 `caseNo` 字段
- 病案首页的 `caseNo` 可能为空
- 没有 fallback 到 `inpatientNo` 的逻辑

### 解决方案
- 使用自定义模板替代直接属性绑定
- 添加 `getCaseNo` 函数实现智能显示
- 病案首页类型优先使用住院号作为病案号

### 改进效果
- ✅ 病案首页的病案号正确显示（使用住院号）
- ✅ 其他类型病案正常显示（使用 caseNo）
- ✅ 向后兼容，不影响现有功能
- ✅ 提供兜底逻辑，避免显示空白
