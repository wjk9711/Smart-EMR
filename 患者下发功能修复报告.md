# 患者下发功能修复报告

## ❌ 问题描述

用户反馈了两个关键问题：

1. **其他用户看不到下发的患者** - 管理员下发患者后，其他用户的患者列表中没有显示收到的患者
2. **重复下发不会创建新副本** - 管理员重复下发同一患者时，系统没有创建新的副本患者，而是将病案插入到相同患者的病案列表中

---

## 🔍 问题分析

### 问题1：其他用户看不到患者

**根本原因**：`PatientAssignment`表中有`copied_patient_id`为null的无效记录。

**诊断结果**：
```sql
-- 发现以下无效记录
SELECT pa.id, pa.patientId, pa.userId, u.username
FROM patient_assignments pa
JOIN users u ON pa.userId = u.id
WHERE pa.copied_patient_id IS NULL;

结果:
- ID=8, patientId=18, userId=11 (student1), copied_patient_id=NULL
- ID=9, patientId=18, userId=13 (teacher001), copied_patient_id=NULL
```

**影响**：
- 查询逻辑使用`copiedPatientId`来获取副本患者ID
- 如果`copied_patient_id`为null，查询条件`id IN (copiedPatientIds)`会过滤掉这些记录
- 导致用户看不到任何患者

**产生原因**：之前的下发操作可能因为某些错误（如字段不存在、事务回滚等）只创建了分配记录，但没有成功创建副本患者。

### 问题2：重复下发不会创建新副本

**根本原因**：`patient_assignments`表有唯一索引`unique_patient_user_assignment`（patientId + userId的组合必须唯一）。

**数据库约束**：
```sql
-- 唯一索引定义
UNIQUE INDEX `unique_patient_user_assignment` (`patientId`, `userId`)
```

**影响**：
- 当尝试为同一个原始患者和同一个用户创建第二条分配记录时
- 数据库抛出`Duplicate entry`错误
- 导致重复下发失败

**原有逻辑**：
```typescript
// 检查是否已经分配过
const existing = await PatientAssignment.findOne({
  where: { 
    patientId: originalPatient.id,
    userId: user.id 
  },
})

if (existing) {
  console.log(`⚠️  患者已分配给用户，跳过`)
  continue // ❌ 直接跳过，不创建新副本
}
```

## 🛠️ 修复方案

### 修复1：清理无效的分配记录

**执行脚本**: `clean-invalid-assignments-final.ts`

**操作**：
```sql
DELETE FROM patient_assignments 
WHERE copied_patient_id IS NULL;
```

**结果**：
```
✅ 删除记录 ID=8 (用户: student1)
✅ 删除记录 ID=9 (用户: teacher001)
✅ 已删除 2 条无效记录
```

### 修复2：移除唯一约束，允许重复下发

#### 2.1 删除数据库唯一索引

**执行脚本**: `remove-assignment-unique-constraint.ts`

**操作**：
```sql
-- 1. 删除唯一索引
ALTER TABLE patient_assignments 
DROP INDEX unique_patient_user_assignment;

-- 2. 添加普通索引（用于查询优化）
CREATE INDEX idx_patient_user 
ON patient_assignments (patientId, userId);
```

**验证**：
```
唯一索引数量: 1
  - PRIMARY (id)

✅ 只有PRIMARY是唯一索引（正确）
```

#### 2.2 更新模型定义

**文件**: `emr-backend/src/models/PatientAssignment.ts`

**修改前**：
```typescript
indexes: [
  {
    fields: ['patientId', 'userId'],
    unique: true,  // ❌ 唯一约束
    name: 'unique_patient_user_assignment',
  },
  // ...
]
```

**修改后**：
```typescript
indexes: [
  {
    fields: ['patientId', 'userId'],
    unique: false,  // ✅ 移除唯一约束，允许重复下发
    name: 'idx_patient_user',
  },
  // ...
]
```

#### 2.3 移除跳过逻辑

**文件**: `emr-backend/src/controllers/inpatientController.ts`

**修改前**：
```typescript
for (const user of users) {
  // 检查是否已经分配过
  const existing = await PatientAssignment.findOne({
    where: {

}
```

**修改后**：
```typescript

  // ✅ 不再检查是否已分配过，允许重复下发
  // 每次下发都创建新的副本患者和病案
  
  // 生成新的唯一住院号
  const newInpatientNo = await generateUniqueInpatientNo()
```

## 💡 关键设计原则

### 1. 完全独立复制

每次下发都创建完全独立的副本：
- ✅ 新的患者ID
- ✅ 新的13位唯一KEY
- ✅ 新的住院号
- ✅ doctorId设置为接收者
- ✅ sourcePatientId=null（无依赖）

### 2. 允许重复下发

移除唯一约束后，可以多次下发同一患者：
- ✅ 每次下发创建新的副本
- ✅ 每个副本有独立的分配记录
- ✅ 用户可以有多个相同名称的患者副本

### 3. 数据隔离

每个副本完全独立：
- ✅ 修改副本A不影响副本B
- ✅ 删除副本A不影响副本B
- ✅ 每个副本有独立的病案
