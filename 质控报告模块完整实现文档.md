# 质控报告模块完整实现文档

## 📋 功能概述

在病历质控模块下新增"质控报告"功能，支持用户创建、编辑、保存Word格式的质控报告。每个报告都有唯一的15位随机关键值用于区分。

---

## ✅ 已完成的功能

### 1. 数据库设计

**表名**: `quality_reports`

**字段说明**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键，自增 |
| user_id | INT | 用户ID（外键） |
| report_key | VARCHAR(15) | 唯一关键值（15位大写字母+数字） |
| report_name | VARCHAR(255) | 报告名称 |
| file_path | VARCHAR(500) | Word文档存储路径 |
| file_size | INT | 文件大小（字节） |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

**索引**：
- `idx_user_id`: 用户ID索引
- `idx_report_key`: 唯一关键值索引（UNIQUE）
- `idx_created_at`: 创建时间索引

#### 核心功能

1. **生成15位唯一关键值**
   ```typescript
   const generateReportKey = (): string => {
     const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789'
     let result = ''
     for (let i = 0; i < 15; i++) {
       result += chars.charAt(Math.floor(Math.random() * chars.length))
     }
     return result
   }
   ```

2. **文件存储**
   - 存储路径：`uploads/quality-reports/{userId}_{reportKey}.docx`
   - 自动创建目录
   - 删除时同步删除文件

3. **权限控制**
   - 所有接口都需要JWT认证
   - 用户只能操作自己的报告

## 🔍 技术细节

### 1. 唯一关键值生成

- 字符集：`ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789`（36个字符）
- 长度：15位
- 组合数：36^15 ≈ 2.2 × 10^23（几乎不会重复）
- 生成后检查数据库确保唯一性

### 2. HTML转Word

使用`docx`库进行转换：

```typescript
import { Document, Packer, Paragraph, TextRun } from 'docx'

const doc = new Document({
  sections: [{
    children: [
      new Paragraph({
        children: [
          new TextRun({ text: '报告内容', size: 24 }),
        ],
      }),
    ],
  }],
})

const blob = await Packer.toBlob(doc)
```

**注意**：当前实现是简化版，只提取纯文本。如需保留格式，建议使用专业库如`html-docx-js`。

### 3. 文件存储策略

- **路径规则**：`{userId}_{reportKey}.docx`
- **优点**：
  - 文件名唯一，不会冲突
  - 通过文件名即可定位用户和报告
  - 便于管理和清理

### 4. 数据关系

```
User (1) ----< (N) QualityReport
```

一个用户可以有多个质控报告，每个报告属于一个用户。

## 📝 代码示例

### 创建报告（前端）

```typescript
import { createQualityReport } from '@/api/qualityReport'

const handleCreate = async () => {
  try {
    const res = await createQualityReport({
      reportName: '我的质控报告',
    })
    console.log('创建成功，报告ID:', res.data.id)
    console.log('报告关键值:', res.data.reportKey)
  } catch (error) {
    console.error('创建失败:', error)
  }
}
```

### 保存Word文档（前端）

```typescript
import { updateQualityReport } from '@/api/qualityReport'
import { Packer } from 'docx'

const handleSave = async (reportId: number, htmlContent: string) => {
  // 转换为DOCX
  const doc = convertHtmlToDocx(htmlContent)
  const blob = await Packer.toBlob(doc)
  
  // 转为base64
  const reader = new FileReader()
  reader.readAsDataURL(blob)
  reader.onloadend = async () => {
    const base64 = (reader.result as string).split(',')[1]
    
    // 保存到服务器
    await updateQualityReport(reportId, {
      content: base64,
    })
  }
}
```

### 获取报告列表（前端）

```typescript
import { getQualityReports } from '@/api/qualityReport'

const fetchReports = async () => {
  const res = await getQualityReports()
  reports.value = res.data

    console.log(`关键值: ${report.reportKey}`)
    console.log(`大小: ${report.fileSize} bytes`)
    console.log(`保存时间: ${report.updatedAt}`)
  })
}
```
