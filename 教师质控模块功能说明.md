# 教师质控模块功能说明

## 概述

为教师用户新增了两个质控相关的子模块：**质控评价**和**小组质控**，用于全面监控和分析学生的病历书写质量。

## 功能模块

### 1. 质控评价 (`/quality/control-evaluation`)

#### 功能描述
展示整体质控情况的可视化仪表盘，包含统计概览、诊断正确率分析和AI智能建议。

#### 界面布局

**上半部分：质控概览信息（4个统计卡片）**
- **完成人数**: 43/43 - 显示已完成质控的学生数量
- **平均时间**: 4m21s - 学生完成病历的平均用时
- **平均正确率**: 91.2% - 诊断编码的平均正确率
- **AI分数**: 93 - AI综合评分

**下半部分左侧：诊断正确率图表**
- 使用ECharts柱状图展示各诊断的编码正确率
- 横坐标：诊断名称及ICD-10编码
  - 瘢痕子宫N85.805
  - 梗阻性分娩O66.901
  - 孕39周O26.900x506
  - 孕2次O26.900x603
  - 产2次O26.900x703
  - 胎盘粘连伴出血O72.001
  - 妊娠合并羊水过少O41.001
  - 女性盆腔炎N73.901
  - 轻度贫血D64.906
  - 低钾血症E87.601
  - 真菌性阴道炎B37.304
- 纵坐标：正确率百分比（80%-100%）
- 红色虚线标记平均正确率（91.2%）
- 高于平均值的柱子显示绿色，低于平均值的显示橙色

**下半部分右侧：智能修改建议**
- 基于AI分析生成的针对性改进建议
- 每条建议包含：
  - 优先级标签（重要/一般/提示）
  - 相关诊断名称
  - 详细的改进建议文本
  - AI置信度百分比

#### 技术实现
- 使用Vue 3 Composition API
- ECharts 5.4.3 数据可视化
- Element Plus UI组件库
- 响应式布局设计（支持移动端）

### 2. 小组质控 (`/quality/group-control`)

#### 功能描述
展示六个学习小组的质控情况，通过卡片形式直观呈现各小组的表现，点击卡片可查看详细的质控报告和AI建议。

#### 界面布局

**主页面：小组卡片网格**
- 6个小组卡片，采用响应式网格布局
- 每个卡片包含：
  - 小组名称和头像（渐变色背景）
  - 成员数量
  - 质控分数（大字号突出显示）
  - 进度条可视化
  - 统计数据（完成数、正确率）
  - "查看报告"按钮

**弹窗：详细质控报告**
分为左右两栏：

**左侧：质控报告详情**
1. **基本信息区域**
   - 小组名称、成员数量
   - 质控周期、质控状态（优秀/良好/需改进）

2. **质控统计区域**
   - 总病历数、已完成数、待完善数
   - 诊断统计表：
     - 诊断名称
     - 出现次数
     - 编码正确率
     - 问题数量

3. **常见问题分析**
   - 时间轴展示历史问题
   - 每个问题包含：
     - 问题标题和描述
     - 严重程度标签（严重/中等/轻微）
     - 出现频率

**右侧：AI智能建议**
1. **总体评价**
   - AI对小组整体表现的评估
   - 三个维度的评分：
     - 诊断准确性
     - 编码规范性
     - 记录完整性

2. **改进建议**
   - 可折叠的建议列表
   - 每条建议包含：
     - 建议标题
     - 详细描述
     - 优先级标签
     - 预期提升分数

3. **学习资源推荐**
   - 相关文档、视频课程、学习资料链接
   - 资源类型标签

#### 示例数据

**第一小组**（94.5分 - 优秀）
- 成员：7人
- 完成数：42/45
- 正确率：92.8%
- 主要问题：诊断编码填写不规范、现病史描述不够详细

**第二小组**（91.2分 - 良好）
- 成员：8人
- 完成数：38/42
- 正确率：90.5%
- 主要问题：诊断依据不充分

**第三小组**（88.7分 - 需改进）
- 成员：6人
- 完成数：35/40
- 正确率：87.9%

**第四至六小组**：分数在89-93分之间，表现稳定

## 访问权限

- **教师角色**：可以访问所有质控功能
- **管理员角色**：可以访问所有质控功能
- **学生角色**：无权访问（路由守卫拦截）

## 导航入口

在质控首页（`/quality`）顶部添加了两个按钮：
- **质控评价**（蓝色按钮）→ `/quality/control-evaluation`
- **小组质控**（绿色按钮）→ `/quality/group-control`

## 启动测试

1. 确保依赖已安装：
   ```bash
   cd emr-frontend
   npm install
   ```

2. 启动开发服务器：
   ```bash
   npm run dev
   ```

3. 访问路径：
   - 质控首页: `http://localhost:5173/quality`
   - 质控评价: `http://localhost:5173/quality/control-evaluation`
   - 小组质控: `http://localhost:5173/quality/group-control`

4. 使用教师或管理员账号登录测试
