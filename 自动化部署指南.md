# EMR系统自动化部署与更新指南

## 概述

本项目提供了一套完整的自动化部署和更新脚本，可以从GitHub仓库 `https://github.com/wjk9711/Smart-EMR` 一键拉取代码、构建并部署。

## 脚本说明

### 1. auto-update.sh（完整部署脚本）

**用途：** 首次部署或完整更新服务器

**功能：**
- ✅ 检查系统依赖（Node.js, npm, Git等）
- ✅ 自动备份当前版本
- ✅ 从GitHub拉取最新代码
- ✅ 恢复配置文件（.env等）
- ✅ 构建后端和前端
- ✅ 停止旧服务，启动新服务
- ✅ 配置Nginx反向代理（如果已安装）
- ✅ 配置防火墙规则
- ✅ 验证部署是否成功
- ✅ 清理临时文件

**使用方法：**
```bash
# 上传到服务器
scp auto-update.sh root@47.97.200.237:/root/

# SSH登录服务器
ssh root@47.97.200.237

# 赋予执行权限
chmod +x auto-update.sh

# 运行脚本（需要root权限）
sudo bash auto-update.sh
```

**注意事项：**
- 需要root权限
- 首次运行会创建 `/opt/emr-system` 目录
- 会自动备份到 `/opt/emr-backup`
- 日志保存在 `/var/log/emr-update.log`

---

### 2. quick-update.sh（快速更新脚本）

**用途：** 日常快速更新（适用于已部署环境）

**功能：**
- ✅ 拉取最新代码
- ✅ 停止服务
- ✅ 重新构建前后端
- ✅ 部署前端文件
- ✅ 重启服务
- ✅ 验证服务状态

**使用方法：**
```bash
# SSH登录服务器
ssh root@47.97.200.237

# 进入安装目录
cd /opt/emr-system

# 运行快速更新
bash quick-update.sh
```

**注意事项：**
- 要求已经通过 `auto-update.sh` 完成首次部署
- 不会备份配置文件
- 速度更快，适合频繁更新

---

### 3. update-local.bat（Windows本地更新）

**用途：** 在Windows本地开发环境更新代码

**功能：**
- ✅ 拉取最新代码
- ✅ 停止本地服务
- ✅ 重新构建前后端
- ✅ 启动后端服务

**使用方法：**
```bash
# 在项目根目录双击运行
update-local.bat

# 或在命令行中运行
.\update-local.bat
```

**注意事项：**
- 仅用于本地开发环境
- 不会部署到Web服务器
- 前端需要通过 `npm run dev` 单独启动

## 首次部署流程

### 步骤1：准备服务器

```bash
# SSH登录阿里云服务器
ssh root@47.97.200.237

# 更新系统
apt update && apt upgrade -y  # Ubuntu/Debian
# 或
yum update -y  # CentOS/RHEL

# 安装必要软件
apt install -y git nodejs npm curl nginx  # Ubuntu/Debian
# 或
yum install -y git nodejs npm curl nginx  # CentOS/RHEL

# 安装PM2（可选，用于进程管理）
npm install -g pm2
```

### 步骤2：上传部署脚本

```bash
# 从本地电脑上传
scp auto-update.sh quick-update.sh root@47.97.200.237:/root/
```

### 步骤3：运行完整部署

```bash
# 在服务器上
cd /root
chmod +x auto-update.sh
sudo bash auto-update.sh
```

脚本会：
1. 询问确认
2. 检查依赖
3. 克隆代码到 `/opt/emr-system`
4. 安装依赖并构建
5. 配置Nginx
6. 启动服务
7. 显示访问地址

### 步骤4：配置数据库

```bash
# 编辑后端配置
cd /opt/emr-system/emr-backend
nano .env

# 修改数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=emr_system
DB_USER=root
DB_PASSWORD=你的密码
JWT_SECRET=随机字符串
```

### 步骤5：初始化数据库

```bash
cd /opt/emr-system/emr-backend
npm run db:migrate
npm run db:seed
```

### 步骤6：验证部署

```bash
# 测试健康检查
curl http://localhost:3000/health

# 获取公网IP
curl ifconfig.me

# 浏览器访问
# http://你的公网IP
```

## 日常更新流程

### 方法1：使用快速更新脚本（推荐）

```bash
# SSH登录服务器
ssh root@47.97.200.237

# 运行快速更新
cd /opt/emr-system
bash quick-update.sh
```

### 方法2：手动更新

```bash
# 1. 停止服务
pkill -f "node.*dist/app.js"

# 2. 拉取代码
cd /opt/emr-system
git pull origin main

# 3. 构建后端
cd emr-backend
npm install --production
npm run build

# 4. 构建前端
cd ../emr-frontend
npm install
npm run build

# 5. 部署前端
rm -rf /var/www/html/*
cp -r dist/* /var/www/html/

# 6. 启动服务
cd ../emr-backend
nohup node dist/app.js > backend.log 2>&1 &
```

## 配置说明

### 后端环境变量 (.env)

```bash
# 服务器端口
PORT=3000

# 运行环境
NODE_ENV=production

# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_NAME=emr_system
DB_USER=root
DB_PASSWORD=your_password

# JWT配置
JWT_SECRET=change-this-to-random-string
JWT_EXPIRES_IN=24h

# 文件上传
UPLOAD_DIR=./uploads
MAX_FILE_SIZE=10485760

# 通义千问API（可选）
QWEN_API_KEY=your-api-key
```

### 前端环境变量 (.env.production)

```bash
# API基础URL
# 如果使用Nginx，留空即可
VITE_API_BASE_URL=

# 如果不使用Nginx，填写完整地址
# VITE_API_BASE_URL=http://47.97.200.237:3000
```

### Nginx配置

脚本会自动生成以下配置：

```nginx
server {
    listen 80;
    server_name _;

    # 前端静态文件
    location / {
        root /var/www/html;
        try_files $uri $uri/ /index.html;
        index index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://localhost:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    
    client_max_body_size 10m;
}
```

## 常见问题

### 1. 脚本执行权限不足

```bash
chmod +x auto-update.sh quick-update.sh
sudo bash auto-update.sh
```

### 2. Git拉取失败

```bash
# 检查Git配置
git config --global user.email "your@email.com"
git config --global user.name "Your Name"

# 重新设置远程仓库
cd /opt/emr-system
git remote set-url origin https://github.com/wjk9711/Smart-EMR.git
git pull origin main
```

### 3. npm install 失败

```bash
# 清除缓存
npm cache clean --force

# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 重新安装
npm install
```

### 4. 端口被占用

```bash
# 查找占用3000端口的进程
lsof -i :3000
netstat -tlnp | grep 3000

# 杀死进程
kill -9 <PID>

# 或修改端口
echo "PORT=3001" >> emr-backend/.env
```

### 5. 服务启动失败

```bash
# 查看后端日志
tail -f /opt/emr-system/emr-backend/backend.log

# 查看系统日志
journalctl -u nginx -f

# 检查数据库连接
mysql -u root -p -e "SHOW DATABASES;"
```

### 6. 前端页面空白

```bash
# 检查Nginx配置
nginx -t
systemctl status nginx

# 检查前端文件
ls -la /var/www/html/

# 重新部署前端
cd /opt/emr-system/emr-frontend
npm run build
rm -rf /var/www/html/*
cp -r dist/* /var/www/html/
```

### 7. CORS错误

确保Nginx配置中包含正确的代理头：

```nginx
location /api/ {
    proxy_pass http://localhost:3000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```

## 备份与恢复

### 自动备份

`auto-update.sh` 会自动备份到 `/opt/emr-backup`，保留最近5个版本。

### 手动备份

```bash
# 备份整个系统
tar -czf emr-backup-$(date +%Y%m%d).tar.gz /opt/emr-system

# 备份数据库
mysqldump -u root -p emr_system > emr-db-backup-$(date +%Y%m%d).sql
```

### 恢复备份

```bash
# 停止服务
pkill -f "node.*dist/app.js"

# 恢复代码
tar -xzf emr-backup-20240101.tar.gz -C /

# 恢复数据库
mysql -u root -p emr_system < emr-db-backup-20240101.sql

# 启动服务
cd /opt/emr-system/emr-backend
nohup node dist/app.js > backend.log 2>&1 &
```

## 监控与维护

### 查看服务状态

```bash
# 检查后端进程
ps aux | grep node

# 检查端口监听
netstat -tlnp | grep 3000

# 查看后端日志
tail -f /opt/emr-system/emr-backend/backend.log

# 查看Nginx日志
tail -f /var/log/nginx/access.log
tail -f /var/log/nginx/error.log
```

### 性能优化

```bash
# 使用PM2管理进程（推荐）
npm install -g pm2
cd /opt/emr-system/emr-backend
pm2 start dist/app.js --name emr-backend
pm2 save
pm2 startup

# 查看进程状态
pm2 status
pm2 logs emr-backend
```

### 定时更新

可以设置cron任务定期更新：

```bash
# 编辑crontab
crontab -e

# 每天凌晨3点自动更新（谨慎使用）
0 3 * * * cd /opt/emr-system && bash quick-update.sh >> /var/log/emr-auto-update.log 2>&1
```

## 安全建议

1. **修改默认密码**
   - 登录后立即修改admin密码
   - 修改数据库root密码

2. **配置防火墙**
   ```bash
   ufw allow 80/tcp
   ufw allow 443/tcp
   ufw allow 3000/tcp
   ufw enable
   ```

3. **启用HTTPS**
   ```bash
   # 使用Let's Encrypt
   apt install certbot python3-certbot-nginx
   certbot --nginx -d your-domain.com
   ```

4. **定期备份**
   - 代码已通过Git管理
   - 定期备份数据库
   - 备份重要配置文件

5. **监控日志**
   - 定期检查后端日志
   - 监控Nginx访问日志
   - 设置日志轮转
