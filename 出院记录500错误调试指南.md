## 📊 常见错误和解决方案

### 错误1：唯一约束冲突

**错误信息**：
```
❌ Error name: SequelizeUniqueConstraintError
❌ Error message: caseNo must be unique
```

**原因**：该患者已经有一个出院记录了

**解决方案**：
1. 在病案列表中查找该患者的出院记录
2. 点击"编辑"而不是"新建"
3. 或者删除已有的出院记录后再新建

---

### 错误2：验证错误

**错误信息**：
```
❌ Error name: SequelizeValidationError
❌ Error message: recordType cannot be null
```

**原因**：缺少必填字段

**解决方案**：
- 检查前端是否正确传递了 `recordType: 'discharge'`
- 检查 `patientId` 是否有值
- 检查 `content` 是否为有效的 JSON 字符串

---

### 错误3：外键约束错误

**错误信息**：
```
❌ Error name: SequelizeForeignKeyConstraintError
❌ Error message: Cannot add or update a child row: a foreign key constraint fails
```

**原因**：`patientId` 或 `doctorId` 不存在

**解决方案**：
- 确认患者存在
- 确认当前用户（医生）存在
- 检查 `patientId` 是否正确

---

### 错误4：ENUM 值无效

**错误信息**：
```
❌ Error name: SequelizeDatabaseError
❌ Error message: Incorrect ENUM value: 'discharge'
```

**原因**：数据库中的 ENUM 没有包含 'discharge'

**解决方案**：
1. 运行数据库迁移：`npm run db:migrate`
2. 重启后端服务
3. 如果仍然失败，手动更新数据库：
   ```sql
   ALTER TABLE inpatient_records 
   MODIFY COLUMN recordType ENUM('admission', 'progress', 'discharge', 'operation', 'home_page', 'other') NOT NULL;
   ```
