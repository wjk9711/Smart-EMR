# 首次病程记录提交错误修复报告

## 📋 问题描述

用户反馈：**提交首次病程记录时提示服务器错误，提交失败**

---

## 🔍 问题分析

### 根本原因

首次病程记录表单存在与入院记录相同的问题：

1. **异步加载顺序错误** ❌
   - 编辑时直接 `Object.assign(formData, props.recordData)`
   - 没有等待患者信息加载完成就加载病案数据
   - 导致患者基本信息被覆盖或丢失

2. **患者信息被病案数据覆盖** ❌
   - `loadRecordData` 使用 `Object.keys(progressRecord).forEach` 遍历所有字段
   - 如果病案数据中包含患者字段，会覆盖从患者表加载的信息

3. **编辑时触发唯一约束验证错误** ❌
   - 编辑时发送了 `patientId` 和 `caseNo`
   - `caseNo` 有唯一约束，更新时触发 Sequelize Validation Error

---

## ✅ 解决方案

### 1. 修复异步加载顺序

**修改前** ❌：
```typescript
watch(() => props.visible, async (val) => {
  if (val && props.recordData) {
    isEdit.value = true
    Object.assign(formData, props.recordData)  // ❌ 直接覆盖
    
    if (props.recordData.content && typeof props.recordData.content === 'string') {
      try {
        const content = JSON.parse(props.recordData.content)
        if (content.progressRecord) {
          Object.assign(formData, content.progressRecord)  // ❌ 再次覆盖
        }
      } catch (e) {
        console.error('Failed to parse record content:', e)
      }
    }
  }
})
```

**修改后** ✅：
```typescript
watch(() => props.visible, async (val) => {
  dialogVisible.value = val
  
  if (val && props.recordData) {
    // 编辑模式
    isEdit.value = true
    patientInfoLoaded.value = false
    
    // ✅ 先加载患者信息
    if (props.patientId) {
      await loadPatientInfo(props.patientId)
    }
    
    // ✅ 再加载病案数据（不会覆盖患者信息）
    loadRecordData()
  } else if (val) {
    // 新增模式
    isEdit.value = false
    patientInfoLoaded.value = false
    resetForm()
    
    await nextTick()
    
    if (props.patientId) {
      formData.patientId = props.patientId
      // ✅ 新增时也加载患者信息
      await loadPatientInfo(props.patientId)
    }
  }
})
```

### 2. 增强患者信息加载逻辑

**修改前** ❌：
```typescript
const loadPatientInfo = async (patientId: number) => {
  try {
    const patient: any = await getInpatientPatient(patientId)
    
    if (patient) {
      Object.assign(formData, {  // ❌ 可能被后续操作覆盖
        patientName: patient.name || '',
        gender: patient.gender || '男',
        // ...
      })
      
      patientInfoLoaded.value = true
      await nextTick()
    }
  } catch (error: any) {
    console.error('Failed to load patient info:', error)
    ElMessage.warning('加载患者信息失败')
  }
}
```

**修改后** ✅：
```typescript
const loadPatientInfo = async (patientId: number) => {
  try {
    console.log('开始加载患者信息，patientId:', patientId)
    const patient: any = await getInpatientPatient(patientId)
    
    console.log('患者信息响应:', patient)
    
    if (patient) {
      // ✅ 只更新患者基本信息字段
      const patientFields = [
        'patientName', 'gender', 'age', 'inpatientNo', 
        'department', 'bedNo', 'admissionDate', 'caseNo'
      ]
      
      patientFields.forEach(field => {
        let value: any
        switch (field) {
          case 'patientName':
            value = patient.name || ''
            break
          case 'gender':
            value = patient.gender || '男'
            break
          case 'age':
            value = patient.age ? String(patient.age) : ''
            break
          case 'inpatientNo':
            value = patient.inpatientNo || ''
            break
          case 'department':
            value = patient.department || ''
            break
          case 'bedNo':
            value = patient.bedNo || ''
            break
          case 'admissionDate':
            value = patient.admissionDate || new Date().toISOString().split('T')[0]
            break
          case 'caseNo':
            value = patient.inpatientNo || ''
            break
          default:
            value = ''
        }
        ;(formData as any)[field] = value
      })
      
      patientInfoLoaded.value = true
      console.log('患者信息加载完成')
      await nextTick()
    } else {
      ElMessage.warning('未找到患者信息')
    }
  } catch (error: any) {
    console.error('Failed to load patient info:', error)
    ElMessage.warning('加载患者信息失败')
  }
}
```

### 3. 添加字段保护机制

**新增函数** ✅：
```typescript
// 加载已有病案数据
const loadRecordData = () => {
  if (props.recordData?.content) {
    try {
      const content = typeof props.recordData.content === 'string' 
        ? JSON.parse(props.recordData.content) 
        : props.recordData.content
      
      const progressRecord = content.progressRecord || {}
      
      // ✅ 定义不应该被病案数据覆盖的患者基本信息字段
      const patientInfoFields = [
        'patientName', 'gender', 'age', 'inpatientNo', 
        'department', 'bedNo', 'admissionDate', 'caseNo'
      ]
      
      // ✅ 填充病案特有字段（排除患者基本信息）
      Object.keys(progressRecord).forEach(key => {
        if (!patientInfoFields.includes(key) && progressRecord[key] !== undefined) {
          (formData as any)[key] = progressRecord[key]
        }
      })
      
      // ✅ 同时更新顶层字段（recordDateTime, status等）
      if (props.recordData.recordDateTime) {
        formData.recordDateTime = props.recordData.recordDateTime
      }
      if (props.recordData.status) {
        formData.status = props.recordData.status
      }
      
      console.log('病案数据加载完成')
    } catch (error) {
      console.error('Failed to parse record content:', error)
    }
  }
}
```

### 4. 修复提交时的唯一约束错误

**修改前** ❌：
```typescript
const submitData = {
  patientId: formData.patientId || props.patientId,  // ❌ 编辑时不应更新
  caseNo: formData.caseNo,                            // ❌ 有唯一约束
  recordType: 'progress',
  content: JSON.stringify(recordContent),
  status: formData.status,
}

if (isEdit.value) {
  await updateInpatientRecord(formData.id!, submitData)  // ❌ 触发 Validation Error
}
```

**修改后** ✅：
```typescript
// ✅ 编辑模式：只更新必要字段，避免触发唯一约束
const submitData = isEdit.value 
  ? {
      recordType: 'progress',
      content: JSON.stringify(recordContent),
      status: formData.status,
    }
  : {
      patientId: formData.patientId || props.patientId,
      caseNo: formData.caseNo,
      recordType: 'progress',
      content: JSON.stringify(recordContent),
      status: formData.status,
    }

console.log('提交数据:', submitData)

if (isEdit.value) {
  await updateInpatientRecord(formData.id!, submitData)  // ✅ 只更新必要字段
  ElMessage.success('更新成功')
} else {
  await createInpatientRecord(submitData)
  ElMessage.success('创建成功')
}
```

## ✅ 总结

本次修复解决了首次病程记录提交的服务器错误问题，主要工作包括：

1. ✅ **修复异步加载顺序**：先加载患者信息，再加载病案数据
2. ✅ **添加字段保护机制**：防止患者基本信息被病案数据覆盖
3. ✅ **优化提交逻辑**：编辑时只更新必要字段，避免触发唯一约束
4. ✅ **增强错误日志**：便于问题排查和调试

**核心原则**：
- 患者基本信息自动填充且不可修改
- 确保异步加载顺序正确（先患者信息，后病案数据）
- 保护关键字段不被意外覆盖
- 编辑时只更新必要字段

**技术要点**：
- Vue 3 Composition API（async/await）
- 字段白名单保护机制
- 条件提交数据
- 详细的调试日志
