# EMR系统生产环境部署指南

## 问题诊断

当前错误：`POST http://47.97.200.237/api/auth/login 404 (Not Found)`

**原因分析：**
- 前端构建后直接请求服务器的 `/api` 路径
- Vite的代理配置只在开发环境有效，生产环境无效
- 需要确保后端服务正确运行并能处理 `/api` 路径的请求

## 解决方案

### 方案1：使用环境变量配置API地址（已实施）

#### 前端配置

1. **环境变量文件已创建：**
   - `.env.development` - 开发环境使用 `http://localhost:3000`
   - `.env.production` - 生产环境使用 `http://47.97.200.237:3000`

2. **重新构建前端：**
```bash
cd emr-frontend
npm run build
```

3. **将构建后的文件部署到服务器**

#### 后端配置

1. **确保后端在服务器上运行：**
```bash
cd emr-backend
npm install
npm run build
node dist/app.js
```

2. **配置后端监听所有网络接口：**

修改 `emr-backend/src/app.ts` 中的启动代码，确保监听 `0.0.0.0`：

```typescript
app.listen(config.port, '0.0.0.0', () => {
  console.log(`Server is running on port ${config.port}`)
  console.log(`Environment: ${config.nodeEnv}`)
  console.log(`API URL: http://0.0.0.0:${config.port}/api`)
})
```

3. **配置防火墙/安全组：**
   - 确保服务器的3000端口对外开放
   - 阿里云安全组规则添加入站规则：TCP 3000端口

### 方案2：使用Nginx反向代理（推荐用于生产环境）

如果希望前端和后端在同一域名下，可以使用Nginx：

#### Nginx配置示例：

```nginx
server {
    listen 80;
    server_name 47.97.200.237;

    # 前端静态文件
    location / {
        root /path/to/emr-frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api/ {
        proxy_pass http://localhost:3000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 方案3：修改前端baseURL为完整URL

如果不想使用Nginx，可以修改前端配置直接请求后端完整URL：

在 `.env.production` 中：
```env
VITE_API_BASE_URL=http://47.97.200.237:3000
```

## 常见问题排查

### 1. 后端服务未启动
```bash
# 检查进程
ps aux | grep node

# 启动后端
cd /path/to/emr-backend
npm run start
```

### 2. 端口被占用
```bash
# 查看3000端口占用
lsof -i :3000

# 或使用其他端口
PORT=3001 npm run start
```

### 3. 防火墙阻止
```bash
# Ubuntu/Debian
sudo ufw allow 3000/tcp

# CentOS/RHEL
sudo firewall-cmd --add-port=3000/tcp --permanent
sudo firewall-cmd --reload

# 阿里云：在控制台配置安全组规则
```

### 4. CORS问题
如果前端和后端在不同域名/端口，确保后端CORS配置正确：

在 `emr-backend/src/app.ts` 中：
```typescript
app.use(cors({
  origin: ['http://47.97.200.237', 'http://your-domain.com'],
  credentials: true
}))
```

## 推荐的长期方案

对于生产环境，强烈建议使用 **Nginx反向代理** 方案：
- ✅ 统一域名，避免跨域问题
- ✅ 更好的性能（静态文件由Nginx直接提供）
- ✅ SSL/TLS支持
- ✅ 负载均衡能力
- ✅ 更安全（隐藏后端端口）
