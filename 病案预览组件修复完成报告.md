## 🔍 问题根源

### Vue 运行时不支持 template 字符串编译

错误信息：
```
Component provided template option but runtime compilation is not supported 
in this build of Vue. Configure your bundler to alias "vue" to 
"vue/dist/vue.esm-bundler.js".
```

**原因**：
- 使用了 `defineComponent` + `template` 字符串的方式创建内部组件
- Vue 的运行时构建（runtime-only）不支持动态编译模板字符串
- 需要使用 Vite 的单文件组件（SFC）方式

---

## ✅ 解决方案

### 方案：将内容直接内联到主模板中

**修改前** ❌：
```vue
<template>
  <div v-if="mode === 'embedded'">
    <PreviewContent :data="data" />  <!-- ❌ 使用子组件 -->
  </div>
</template>

<script setup>
const PreviewContent = defineComponent({
  props: ['data'],
  template: `...`  // ❌ template 字符串不被支持
})
</script>
```

**修改后** ✅：
```vue
<template>
  <div v-if="mode === 'embedded'">
    <!-- ✅ 直接在主模板中渲染 -->
    <div v-if="data && Object.keys(data).length > 0">
      <div class="page-title">标题</div>
      <div class="info-grid">...</div>
      ...
    </div>
  </div>
</template>

<script setup>
// ✅ 不需要 PreviewContent 组件
const data = computed(() => { ... })
</script>
```

## 🎯 技术要点

### 1. 双模式支持

所有预览组件都支持两种显示模式：

```typescript
interface Props {
  visible: boolean
  recordData?: any
  mode?: 'dialog' | 'embedded'  // 显示模式
}
```

- **dialog 模式**：用于单个病案预览，带对话框
- **embedded 模式**：用于统一预览列表，直接显示内容

### 2. 数据提取

使用 `computed` 确保响应式：

```typescript
const data = computed(() => {
  if (!props.recordData) return {}
  
  let content = props.recordData.content
  if (typeof content === 'string') {
    try {
      content = JSON.parse(content)
    } catch (e) {
      console.error('Failed to parse content:', e)
      return {}
    }
  }
  
  // 根据病案类型提取对应的数据
  return content.progressRecord || content || {}
})
```

### 3. 空数据处理

添加友好的空数据提示：

```vue
<div v-if="data && Object.keys(data).length > 0">
  <!-- 正常内容 -->
</div>
<div v-else style="text-align: center; padding: 40px; color: #999;">
  暂无病案数据
</div>
```

### 4. 打印功能

每个组件都实现了独立的打印功能：

```typescript
const handlePrint = () => {
  if (!printArea.value) return
  
  const printWindow = window.open('', '_blank')
  if (printWindow) {
    printWindow.document.write(`
      <!DOCTYPE html>
      <html>
      <head>
        <style>
          /* 打印样式 */
        </style>
      </head>
      <body>
        ${printArea.value.innerHTML}
      </body>
      </html>
    `)
    printWindow.document.close()
    printWindow.print()
  }
}
```

## 💡 经验教训

### 1. Vue 运行时限制

- **Runtime-only 构建**不支持 `template` 字符串
- 必须使用 Vite 的 SFC（单文件组件）方式
- 或者配置别名使用完整版 Vue

### 2. 组件设计原则

- **避免过度嵌套**：简单的预览功能不需要拆分子组件
- **保持单一职责**：每个组件只负责一种病案类型的预览
- **复用通用逻辑**：格式化函数可以提取为工具函数

### 3. 调试技巧

- 使用 `console.log` 输出关键数据
- 检查 `data` computed 是否正确提取数据
- 使用 Vue Devtools 检查组件状态
