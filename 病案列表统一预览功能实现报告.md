## 🎯 功能需求

**用户需求**：在病案列表中，不再需要在每个病案后的操作进行查看，而是在新增病案左侧增加一个预览功能，可以同时预览所有类型的病案，预览顺序按照住院病案首页、入院记录、首次病程记录、手术记录、出院记录。若病案列表中无对应类型的病案，则不加入预览。

### 3. 核心逻辑

#### 数据变量

```typescript
// 全部病案预览相关
const previewAllVisible = ref(false)           // 预览对话框显示状态
const previewRecords = ref<any[]>([])          // 预览病案列表（已排序）
const currentPreviewIndex = ref(0)             // 当前预览索引
const currentPreviewUrl = ref('')              // iframe 预览 URL

// 计算属性：当前预览的病案记录
const currentPreviewRecord = computed(() => {
  if (previewRecords.value.length > 0 && currentPreviewIndex.value >= 0) {
    return previewRecords.value[currentPreviewIndex.value]
  }
  return null
})
```

#### 预览全部病案函数

```typescript
const handlePreviewAll = () => {
  // 定义预览顺序：病案首页、入院记录、首次病程记录、手术记录、出院记录
  const typeOrder = ['home_page', 'admission', 'progress', 'operation', 'discharge']
  
  // 按类型分组
  const recordsByType: Record<string, any[]> = {}
  recordList.value.forEach(record => {
    if (!recordsByType[record.recordType]) {
      recordsByType[record.recordType] = []
    }
    recordsByType[record.recordType].push(record)
  })
  
  // 按顺序合并
  previewRecords.value = []
  typeOrder.forEach(type => {
    if (recordsByType[type] && recordsByType[type].length > 0) {
      previewRecords.value.push(...recordsByType[type])
    }
  })
  
  if (previewRecords.value.length === 0) {
    ElMessage.warning('暂无可预览的病案')
    return
  }
  
  currentPreviewIndex.value = 0
  previewAllVisible.value = true
}
```

**执行流程**：
1. 定义预览顺序数组 `typeOrder`
2. 遍历 `recordList`，按 `recordType` 分组
3. 按照 `typeOrder` 的顺序，依次将各类型的病案添加到 `previewRecords`
4. 如果某个类型没有病案，则跳过
5. 打开预览对话框，从第一个病案开始预览

#### 关闭预览函数

```typescript
const handleClosePreview = () => {
  previewRecords.value = []
  currentPreviewIndex.value = 0
  currentPreviewUrl.value = ''
}
```

**作用**：清理预览数据，释放内存

#### 上一个/下一个病案

```typescript
// 上一个病案
const handlePrevRecord = () => {
  if (currentPreviewIndex.value > 0) {
    currentPreviewIndex.value--
  }
}

// 下一个病案
const handleNextRecord = () => {
  if (currentPreviewIndex.value < previewRecords.value.length - 1) {
    currentPreviewIndex.value++
  }
}
```

## 📊 功能特性

### 1. 智能排序

**预览顺序**：
1. 病案首页（home_page）
2. 入院记录（admission）
3. 首次病程记录（progress）
4. 手术记录（operation）
5. 出院记录（discharge）

**示例**：
```
病案列表（原始顺序）：
- 入院记录 A
- 出院记录 B
- 病案首页 C
- 手术记录 D

预览顺序（自动排序）：
1. 病案首页 C
2. 入院记录 A
3. 手术记录 D
4. 出院记录 B
```

### 2. 动态过滤

**规则**：只包含存在的病案类型

**示例**：
```
场景1：只有病案首页和出院记录
预览列表：[病案首页, 出院记录]

场景2：五种类型都有
预览列表：[病案首页, 入院记录, 首次病程记录, 手术记录, 出院记录]

场景3：没有任何病案
提示："暂无可预览的病案"
```

### 3. 便捷导航

**三种导航方式**：
1. **左侧导航栏点击**：直接跳转到指定病案
2. **"上一个"按钮**：切换到前一个病案
3. **"下一个"按钮**：切换到后一个病案

**视觉效果**：
- 当前选中的病案高亮显示（蓝色背景 + 左边框）
- 鼠标悬停时显示浅灰色背景
- 到达边界时按钮自动禁用

## 💡 技术要点

### 1. 病案类型分组算法

```typescript
// 按类型分组
const recordsByType: Record<string, any[]> = {}
recordList.value.forEach(record => {
  if (!recordsByType[record.recordType]) {
    recordsByType[record.recordType] = []
  }
  recordsByType[record.recordType].push(record)
})
```

**时间复杂度**：O(n)，只需遍历一次病案列表

### 2. 计算属性优化

```typescript
const currentPreviewRecord = computed(() => {
  if (previewRecords.value.length > 0 && currentPreviewIndex.value >= 0) {
    return previewRecords.value[currentPreviewIndex.value]
  }
  return null
})
```

**优势**：
- 自动响应 `currentPreviewIndex` 的变化
- 避免手动维护当前病案对象
- 提供空值保护

## ⚠️ 注意事项

### 1. 病案首页预览

当前实现中，病案首页使用 `HomePagePreview` 组件直接渲染：

```vue
<HomePagePreview 
  v-if="currentPreviewRecord.recordType === 'home_page'"
  :visible="true"
  :record-data="currentPreviewRecord"
/>
```

**优点**：
- 无需加载 iframe
- 性能更好
- 样式一致

**注意**：
- 需要确保 `HomePagePreview` 组件支持外部传入 `record-data`
- 组件内部不应依赖弹窗的 visible 状态

### 2. 其他类型病案预览

对于非病案首页类型，当前使用 iframe 加载：

```vue
<iframe 
  v-else
  :src="currentPreviewUrl"
  frameborder="0"
></iframe>
```

**待完善**：
- 需要根据病案 ID 生成预览 URL
- 可以考虑创建统一的预览组件

**建议方案**：
```typescript
// 监听 currentPreviewRecord 变化，更新 URL
watch(currentPreviewRecord, (record) => {
  if (record && record.recordType !== 'home_page') {
    currentPreviewUrl.value = `/inpatient/record/${record.id}?preview=true`
  }
})
```

### 3. 性能考虑

**大数据量场景**：
- 如果病案数量很多（>100），左侧导航栏可能会很长
- 建议添加虚拟滚动或分页

**优化方案**：
```vue
<div class="preview-nav">
  <el-scrollbar max-height="60vh">
    <!-- 导航项 -->
  </el-scrollbar>
</div>
```

### 4. 键盘快捷键（可选增强）

可以添加键盘快捷键提升用户体验：

```typescript
// 监听键盘事件
onMounted(() => {
  document.addEventListener('keydown', handleKeydown)
})

onUnmounted(() => {
  document.removeEventListener('keydown', handleKeydown)
})

const handleKeydown = (e: KeyboardEvent) => {
  if (!previewAllVisible.value) return
  
  if (e.key === 'ArrowLeft') {
    handlePrevRecord()
  } else if (e.key === 'ArrowRight') {
    handleNextRecord()
  } else if (e.key === 'Escape') {
    previewAllVisible.value = false
  }
}
```
